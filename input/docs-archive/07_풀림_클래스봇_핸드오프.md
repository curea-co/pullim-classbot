## 1. 서비스 개요

### 1.1 한 줄 정의
**클래스봇은 선생님이 자신의 수업·교안·목소리를 AI에 이식해 "디지털 분신(分身) 수업 동반자"를 만드는 도구다.**

### 1.2 포지셔닝
| 축 | ClassBot의 위치 |
|---|---|
| 주도권 | 교사 중심 (AI는 조수) |
| 시점 | 수업 시간·수업 전후 |
| 주 사용처 | 교실, 학원, 1:1 과외 |
| 경쟁 차별점 | 챗봇이 아니라 "선생님의 수업 방식을 학습한 디지털 분신" |

### 1.3 타겟 유저
1. **P1 (주) 교사**: 초·중·고 교사, 학원 강사, 1:1 과외 선생님 — 수업 준비·진행·피드백을 자동화하고 싶은 사람
2. **P2 학생**: 교사가 배정한 ClassBot을 수업·과제·질의응답에 활용
3. **P3 학부모**: 자녀 학습 현황·위기 신호를 주간 리포트로 확인
4. **P4 학원 관리자**: 강사별 ClassBot 운영 현황·과목별 학습 성과 집계

### 1.4 핵심 가치 제안 (Value Props)
- **V1 · 수업 복제성**: 한 번의 수업 설계로 N개 반에서 일관된 품질 유지
- **V2 · 24/7 연속성**: 교실 밖에서도 "선생님의 목소리"로 질의응답
- **V3 · 투명한 통제**: 5단계 Scope Guard로 AI가 "무엇을 해도 되는지" 교사가 정의
- **V4 · 데이터 증거**: 모든 상호작용이 학습 데이터로 쌓여 피드백의 근거가 됨
- **V5 · 감정 감지**: 성적뿐 아니라 학생의 번아웃·감정 리스크를 조기 포착

## 2. 디자인 원칙 (Non-negotiable)

1. **교사의 편집권이 AI보다 우선한다.** 모든 AI 출력은 교사가 수정 가능한 초안이다.
2. **스코프 가드 없는 AI 응답은 금지한다.** 모든 봇은 L1-L5 중 하나의 모드로 반드시 실행된다.
3. **가르치는 감각을 흉내내되, 감각을 소유하지 않는다.** 봇은 교사의 톤·리듬·관용구를 학습하되, 교사의 이름으로 발언하지 않는다 (저작권/명예).
4. **학생 데이터 최소 수집 · 최대 보호.** 교실 영상은 저장하지 않고, 음성도 30일 자동 삭제.
5. **모든 평가는 교사 검수 전 초안일 뿐이다.** 자동 채점은 제안이지 판결이 아니다.

## 4. 주요 기능 · 상세

### 4.1 클래스봇 빌더 (8단계 위자드) — 핵심 기능

교사가 새 클래스봇을 만드는 단일 진입 경로.

| # | 단계 | 입력 | 산출물 |
|---|---|---|---|
| 1 | 봇 정체성 | 이름, 과목, 학년, 캐릭터 톤 (정중/친근/스파르타) | 봇 프로필 |
| 2 | 목소리·페르소나 | 음성 샘플 업로드 (선택) 또는 TTS 프리셋 | 목소리 프로필 |
| 3 | 교안 업로드 | PPT, PDF, 손글씨 노트, 기존 수업 녹화 | RAG 인덱스 |
| 4 | 수업 방식 | 강의형/토론형/문제풀이형/혼합 | 기본 응답 전략 |
| 5 | 스코프 가드 | L1-L5 중 기본값 선택 | 권한 모델 |
| 6 | 평가 기준 | 과제 채점 루브릭, 오답 피드백 스타일 | 평가 엔진 설정 |
| 7 | 학생 안전 | 개인정보 필터, 민감 질문 차단 규칙 | 가드레일 |
| 8 | 테스트·배포 | 교사 직접 테스트 → 반·개인 배정 | 배포된 봇 |

**핵심 UX 요구사항**:
- 각 단계 좌측에 진행 바(rail) · 우측에 미리보기
- 1-7 단계는 드래프트 저장 가능, 8단계에서만 "배포"
- "이전 봇 복제" 버튼으로 새 봇 5분 내 생성 가능

### 4.2 스코프 가드 (5-Level Scope Guard)

AI 봇이 "무엇을 해도 되는지" 정의하는 권한 모델.

| 레벨 | 명칭 | 허용 행동 | 차단 행동 | 사용 시나리오 |
|---|---|---|---|---|
| **L1** | 블록 | 응답 자체를 차단 (metadata만) | 모든 AI 응답 | 중간/기말고사 중 |
| **L2** | 경고 | 질문은 받지만 "이 질문은 답할 수 없어요" | 정답·풀이·힌트 | 쪽지시험 |
| **L3** | 개념 | 관련 개념·정의·예시 제공 | 문제 풀이 절차 | 일반 수업 중 |
| **L4** | 단계 | 풀이 방향·단계 제시 | 최종 답 | 숙제 지원 |
| **L5** | 답 | 모든 응답 허용 (정답 포함) | 없음 | 복습·오답 정복 |

**구현 요구사항**:
- 레벨 변경은 교사만 가능 (학생 UI에 변경 버튼 없음)
- 시간대별 자동 레벨 스위치 (예: 수업 시간 L3, 밤 L5)
- 변경 이력은 감사 로그에 영구 기록

### 4.3 3-Tier AI 모델 라우터

비용·성능 최적화를 위한 모델 선택 엔진.

| 티어 | 사용 모델 예 | 용도 | 응답 시간 |
|---|---|---|---|
| **T1** | Haiku 4.5 / Gemini Flash | 단답·FAQ·출석 확인 | <1초 |
| **T2** | Sonnet 4.6 / GPT-4o-mini | 대부분의 문제 풀이 코칭 | 1-3초 |
| **T3** | Opus 4.7 / GPT-5 | 심화 추론, 긴 해설, 루브릭 기반 서술형 채점 | 3-10초 |

**라우팅 규칙**:
- 기본: T2
- 교사가 "심화" 태그 단 질문: T3
- 짧은 문장 (<20자) 응답: T1
- 학생 1인당 월 T3 호출 200회 기본 한도 (초과 시 승인 요청)

### 4.4 실시간 수업 (Live Classroom)

교사가 수업 중 ClassBot을 활용하는 메인 화면.

**화면 구성**:
- 좌측: 학생 명단 + 발언·집중도 히트맵
- 중앙: 수업 진행 UI (퀴즈·화면 공유)
- 우측: "학생이 봇에 물어본 것들" 실시간 피드
- 하단: 즉석 액션 바 (퀴즈 쏘기·폴 생성·핫스팟 마킹)

**핵심 상호작용**:
- 수업 녹화 자동 시작 (학생 음성은 저장 X, 교사 음성만)
- 학생이 봇에 "질문"할 때 교사 화면에 원문 표시 → 교사가 "수업 전체 공유" 토글 가능
- 집중도 저하 감지 시 (3명 이상 5분 무응답) 자동 알림

### 4.5 즉석 퀴즈·폴 (Quiz & Poll)

- 퀴즈 유형: 객관식 / 단답형 / O·X / 드래그매칭
- 생성: "주제 한 줄 + 난이도" 입력 → 30초 내 5문항 초안
- 결과: 실시간 응답 분포, 오답 클러스터링
- 퀴즈 결과는 자동으로 해당 학생의 학습 프로필에 반영

### 4.6 수업 리플레이

수업 영상·음성·퀴즈·채팅을 타임라인으로 재생.

- 시간 축: 0:00-50:00
- 레이어:
  1. 음성 트랜스크립트 (STT, 한국어+영어 혼합)
  2. 퀴즈·폴 시점 마커
  3. 학생 질문 마커
  4. 집중도 히트맵
- 학생은 본인이 응답한 구간만 재접근 가능 (프라이버시)

### 4.7 리포트 센터

개별 학생 / 반 / 과목 단위 리포트 자동 생성.

**리포트 종류**:
1. 일일 리포트 · 학생별 · 당일 학습량·정답률·감정
2. 주간 리포트 · 학부모 전송용 · 8개 지표 요약
3. 단원 완료 리포트 · 교사 수업 종료 시
4. 번아웃 조기 경보 · 임계 초과 시 즉시 푸시

**공통 구성 요소**:
- 8개 KPI (일 학습시간, 정답률 추세, 세션 중단율, 감정 체크인 등)
- Wee센터 연결 버튼 (심리 상담 리소스)
- 학부모 승인 (24-48H 자동 승인)

### 4.8 채점 허브 (Hybrid Grading)

객관식 자동 + 서술형 AI 초안 + 교사 검수 3단계.

| 단계 | 주체 | 산출물 |
|---|---|---|
| 1 | AI (T2) | 객관식 즉시 채점, 서술형 초안 점수+코멘트 |
| 2 | AI (T3) | 상위 10% 분석 (왜 고득점인가) + 하위 10% 오답 패턴 추출 |
| 3 | 교사 | 초안 검수, 20% 이상 변경 시 AI 재학습 제안 |

**데이터 흐름**: 교사 수정 → 스튜디오로 집계 → 루브릭 자동 개선

### 4.9 AI 학습 코치

**사용 주체**: 학생
**기능**: 주간 학습 상담·감정 체크·목표 재설정
**핵심 원칙**: "교사 대신이 아니라 교사를 보조"
- 위기 신호 감지 시 교사에게 자동 알림
- "자살"·"우울" 등 민감 키워드는 즉시 교사+보호자+Wee센터 3자 알림

### 4.10 템플릿 마켓

교사가 만든 봇·교안·퀴즈를 다른 교사에게 판매·공유.

- 무료 템플릿 (커뮤니티 기여)
- 유료 템플릿 (수익 7:3 크리에이터:플랫폼)
- curea 공식 템플릿 (품질 검수 마크)
- 다운로드 시 원저작자 메타데이터 보존

## 5. 핵심 사용자 플로우

### Flow A · 교사가 새 봇을 만들어 수업에 투입 (첫 사용)
```
1. 로그인 → 홈 대시보드
2. "새 클래스봇" 버튼 클릭
3. 8단계 위자드 완료 (평균 15분)
4. 테스트 메시지 5개 보내고 확인
5. 반 선택 → 배포
6. 첫 수업 시작 → 실시간 수업 화면 진입
7. 수업 종료 → 리플레이 자동 생성
8. 일일 리포트 확인
```

### Flow B · 학생이 봇에 질문 (수업 중)
```
1. 학생 앱 접속 → 반 봇 선택
2. "사인법칙이 뭐예요?" 입력
3. 봇이 Scope Guard L3로 응답 (개념만)
4. 학생이 이해 못함 → "더 쉽게" 버튼
5. 봇 재설명 + 교사 화면에 질문 표시
6. 교사가 "전체 공유" → 반 전체에 설명 노출
```

### Flow C · 서술형 과제 채점 사이클
```
1. 교사: 과제 생성 (루브릭 포함)
2. 학생 제출 → AI 초안 채점 (T2)
3. 교사: 채점 허브 진입, 초안 검토
4. 교사가 20% 이상 수정 → 시스템이 "루브릭 재학습" 제안
5. 교사 승인 → AI가 다음 과제부터 개선된 기준 적용
6. 최종 점수 학부모 리포트 반영
```

### Flow D · 번아웃 감지 → 개입
```
1. 시스템이 학생의 감정 체크인 3일 연속 "힘듦" 감지
2. 리포트 센터에 ⚠ 알림 표시
3. 교사 푸시 알림
4. 교사가 1:1 상담 개시 or Wee센터 연결
5. 학부모에게도 요약 전송 (민감 정보 제외)
```

### Flow E · 학원 관리자 운영 대시보드
```
1. 관리자 로그인
2. 강사별 봇 활성도 확인
3. 과목별 학생 정답률 비교
4. 번아웃 위험군 리스트 확인
5. 월간 학원 리포트 자동 생성 → 원장 전달
```

## 6. 유스케이스

### UC-1 · 학원 수학 강사의 반복 수업 자동화
- 페르소나: 김수학 선생님 (대치동, 5개 반 동일 커리큘럼)
- 문제: 매일 같은 개념을 5번 반복 설명 → 에너지 고갈
- 해결: ClassBot이 개념 설명을 담당, 교사는 심화·문답에 집중
- 성과: 수업 준비시간 40% 감소, 학생 1:1 문답 시간 2배

### UC-2 · 공교육 고등학교 영어 교사의 수행평가 분석
- 페르소나: 박영어 선생님 (공립 고2, 180명)
- 문제: 수행평가 180개 서술형 채점이 2주 걸림
- 해결: AI 초안 채점 → 교사는 검수·코멘트 추가
- 성과: 2주 → 3일로 단축, 코멘트 질 상승

### UC-3 · 개인 과외 선생님의 학생 프로파일링
- 페르소나: 이민서 과외쌤 (10명, 수학·영어)
- 문제: 학생별 진도·오답 패턴을 수기로 관리 어려움
- 해결: ClassBot이 매 수업 자동 기록 → 학부모 주간 리포트 전송
- 성과: 수업료 인상 설득력 증가, 학부모 재계약률 향상

### UC-4 · 중학교 담임의 학습 부진아 조기 개입
- 페르소나: 정담임 선생님 (중2 담임)
- 문제: 학습 부진·심리적 위기가 월말 성적표로만 드러남
- 해결: 감정 체크인·정답률 추세 실시간 모니터링
- 성과: 개입 평균 시점 1개월 → 1주일로 단축

### UC-5 · EBS·EDWITH 연계 보충 자료
- 페르소나: 고3 수험생
- 문제: 인강 수업 후 복습 시 질문할 선생님 없음
- 해결: 인강 선생님의 공식 ClassBot이 복습·추가 질문 수용
- 성과: 인강 완강률 상승, 보조 매출 발생

## 7. 핵심 개념·시스템

### 7.1 봇 정체성 (Bot Identity)
- **이름**: 교사가 자유 설정 (예: "수학이 형")
- **페르소나 파일**: JSON, 기본 50개 속성 (말투, 인사말, 칭찬 스타일...)
- **목소리 ID**: ElevenLabs/OpenAI TTS voice clone (교사 동의 필수)
- **RAG 인덱스**: 교사 업로드 자료의 벡터 DB (Pinecone/pgvector)

### 7.2 수업 이벤트 (Lesson Event)
모든 교실 상호작용은 Event 스트림으로 기록.
```
{
  event_id: UUID,
  lesson_id: UUID,
  type: "quiz_answer" | "bot_question" | "attendance" | "emotion_check",
  actor: student_id | teacher_id | bot_id,
  timestamp: ISO8601,
  payload: {...},
  scope_level: 1-5
}
```

### 7.3 웰빙 지수 (Wellbeing Index)
5개 지표의 가중 평균 (0-100):
1. 일 학습 시간 (목표 대비 %)
2. 정답률 추세 (7일 이동 평균)
3. 세션 중단율 (완료/시작 비율)
4. 감정 체크인 (학생 자기보고)
5. 봇 상호작용 빈도 (너무 많거나 너무 적으면 감점)

임계값 아래 3일 지속 → 교사 알림

### 7.4 저작권·페르소나 권리
- 교사의 목소리·이미지·수업 자료는 교사 소유 (플랫폼 라이센스는 비독점)
- 봇 삭제 시 관련 데이터 30일 내 파기 (백업 포함)
- 교사 탈퇴 시 학생 데이터는 학원/학교가 소유권 선택 가능

## 8. 데이터 모델 (핵심 엔티티)

```
Organization (1) ── (N) Classroom (1) ── (N) Enrollment (N) ── (1) Student
Organization (1) ── (N) Teacher
Teacher (1) ── (N) ClassBot
ClassBot (1) ── (N) Lesson (1) ── (N) LessonEvent
Lesson (1) ── (1) Replay (1) ── (N) Segment
ClassBot (1) ── (N) Asset (교안/음성/이미지)
ClassBot (1) ── (1) ScopeProfile (권한 설정)
Student (1) ── (N) Submission (1) ── (N) GradingDraft (1) ── (1) FinalGrade
Student (1) ── (N) EmotionCheckIn
Student (1) ── (N) WellbeingIndex (일별 스냅샷)
Teacher (1) ── (N) Template → Marketplace
Parent (N) ── (N) Student (관계 테이블, 승인 상태)
```

**키 인덱스 권장**:
- `LessonEvent.lesson_id` + `timestamp` (리플레이)
- `Submission.student_id` + `assignment_id`
- `WellbeingIndex.student_id` + `date` (리포트)

## 9. 연동·통합

### 9.1 풀림 내부 서비스
| 서비스 | 연동 방향 | 내용 |
|---|---|---|
| **풀림 플래너** | ClassBot → Planner | 숙제·시험 일정 자동 주입 |
| **풀림 Q** | ClassBot ↔ Q | 생성된 문제를 Q에 배포 · Q 풀이 로그가 리포트로 환원 |
| **풀림 스튜디오** | Studio → ClassBot | 교사가 만든 교안·문제 ClassBot 교안으로 자동 흡수 |
| **풀림 스토어** | Store → ClassBot | 외부 교재 구매 시 ClassBot 지식 베이스에 추가 |

### 9.2 외부 시스템
- **구글 클래스룸**: 출석·과제·성적 양방향 동기화 (v1 필수)
- **네이스(NEIS)**: 공교육 학적부 연동 (v2)
- **카카오톡**: 학부모 알림 (BIZ MESSAGE)
- **Zoom / Google Meet**: 온라인 수업 시 영상·채팅 수집
- **Wee센터 API**: 심리 상담 연계 (정부 기관 연동, v2)

## 13. 리스크·고려사항

### 13.1 법·윤리
- **교사 목소리 복제**: 교사 서면 동의 + 언제든 철회권 (GDPR-style)
- **학생 감정 데이터**: 민감 정보로 분류, 교사·학부모 외 열람 금지
- **미성년 개인정보**: 만 14세 미만은 법정대리인 동의 필수
- **AI 편향성**: 분기별 편향 감사 (성별·지역·계층)

### 13.2 기술
- **LLM 환각**: T3 응답에 "AI 답변, 검증 필요" 뱃지 기본 표시
- **대량 접속**: 수업 시간 동시 10k 학생 접속 대비 로드 설계
- **오프라인 모드**: 학교 네트워크 불안정 대비 12시간 캐시

### 13.3 비즈니스
- **교사 월 구독료**: ₩19,900 (개인) / ₩49,000 (학원 5인) 수준 검토
- **학원 B2B**: 학생 수 기반 정액
- **무료 티어**: 봇 1개, 학생 10명, T2만 — 공교육 포용
