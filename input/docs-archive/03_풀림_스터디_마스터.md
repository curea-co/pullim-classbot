## 1. 전략적 포지셔닝

> **"AI가 나만의 과외 선생님이 되는 시대. 단순히 문제를 푸는 것이 아니라, 학습 과정 전체를 AI가 코칭하는 지능형 학습 플랫폼"**

스터디는 풀림 생태계에서 **학습자가 콘텐츠를 소비하고 학습하는 공간**이다. 동시에 학습 과정에서 생성되는 데이터가 데이터 플라이휠의 **핵심 연료**로서, 스튜디오의 AI 품질 향상과 스토어의 추천 정확도 향상에 직접 기여한다.

### 1.1 비전

하버드 RCT 연구에서 AI 튜터링이 기존 수업 대비 학습 효과를 **2배** 향상시킨다는 결과가 발표되었고(2025), Gartner는 2027년까지 멀티 에이전트 시스템이 70%가 특화 에이전트로 구성될 것으로 전망했다. 풀림 스터디는 이러한 Agentic AI 흐름의 최전선에서, **자율적으로 학습 경로를 설계하고 실행하는 AI 학습 파트너**를 지향한다.

### 1.2 서비스 대상

| 순위 | 타겟 | 설명 |
|------|------|------|
| 1차 | 수능·내신을 준비하는 고등학생 (고1~고3, 재수생) | 핵심 사용자 |
| 2차 | 중학생 (내신 준비), 자기주도 학습자 | 확장 사용자 |
| 3차 | 학원·기관 (학생 관리 도구), 학부모 (학습 현황 모니터링) | B2B/보조 사용자 |

## 2. 설계 철학

### 2.1 네이밍 원칙

v2에서 모든 기능에 **브랜드 네이밍**(풀림 ○○)을 부여하여 학생이 즉시 기능을 이해할 수 있도록 했다.

**1. 행동 기반**: 기술 명칭("적응형 난이도 엔진") 대신 학생의 행동("풀림 무한풀기")을 반영.  
**2. 한 단어 연상**: 기능명만 보고 3초 안에 무엇인지 떠올릴 수 있어야 한다.  
**3. 풀림 패밀리**: 모든 기능은 "풀림 ○○" 형태를 유지하여 브랜드 일관성을 확보.

### 2.2 3대 설계 원칙 (유지)

**원칙 1 — 파이프라인 기반 진입장벽**: CUREA DEEP + 3-Depth 교육과정 분류 + Self-Refine/RLHF 결합으로 단순 LLM API로 복제 불가능.  
**원칙 2 — 플라이휠 내장**: 사용할수록 데이터↑ → 기능 강화 → 사용자↑ 자기강화 루프.  
**원칙 3 — 도메인 확장성**: 수능·내신을 넘어 자격증, 어학, 코딩, 기업교육, 글로벌까지 동일 아키텍처로 확장.

## 4. Core 기능 — 출시 필수 (5개)

### 4.1 풀림 무한풀기 (Infinity Engine) ★★★

> *"문제가 떨어지지 않는다. 내 실력에 딱 맞는 문제가 끊임없이 나온다."*

학생의 실력을 실시간으로 측정하여 최적 난이도의 문제를 자동 선택하고, 적절한 문제가 없으면 CUREA DEEP 엔진이 즉석에서 신 문제를 생성하는 **무한 적응형** 학습 엔진.

#### 학생이 경험하는 흐름
```
[시작] 과목·단원 선택
  → [풀이] 내 수준에 맞는 문제 출제 (너무 쉽지도, 너무 어렵지도 않음)
  → [적응] 맞추면 조금 더 어렵게, 틀리면 조금 더 쉽게
  → [생성] 기존 문제가 없으면 AI가 즉시 생성 (학생은 차이를 모름)
  → [반복] 원하는 만큼 계속 풀 수 있음 → 문제가 끝나지 않음
```

#### IRT 3-Parameter 모델 (3PL)
```
P(θ) = c + (1 - c) / (1 + exp(-a(θ - b)))
• θ: 학생 능력치 (실시간 추정)  • a: 변별도  • b: 난이도  • c: 추측 확률
```

#### 적응형 엔진 플로우
```
[1] 학생 θ 조회 (Redis) → [2] 최적 난이도 범위 계산 (ZPD)
  → [3] 문제 풀 검색 (Pinecone + MongoDB)
  → [4] 적합 문제 있음? → YES → 문제 반환
                         → NO → [5] CUREA DEEP 실시간 생성 요청
  → [6] 풀이 결과 수신 → [7] IRT θ 업데이트 (EAP 추정)
  → [8] Redis에 θ 갱신 + PostgreSQL에 이력 저장
```

#### 기술적 해자

| 해자 요소 | 설명 | 경쟁사 복제 난이도 |
|----------|------|-----------------|
| IRT 3-Parameter 모델 | 문항별 난이도·변별도·추측도 사전 계산 | 중간 |
| ZPD 필터링 | 근접발달영역 내 문제만 선별 | 낮음 |
| **실시간 문제 생성** | 적합한 기존 문제가 없을 때 CUREA DEEP이 즉시 생성 | **매우 높음** |
| **사후 난이도 보정** | 학생 풀이 데이터로 문항 IRT 파라미터를 자동 보정 | **매우 높음** |
| Deep Knowledge Tracing | 단원 간 전이 효과를 모델링하여 연관 개념도 함께 추천 | 높음 |

#### 경쟁사 비교

| 서비스 | 적응형 | 문제 생성 | 무한 문제 풀 |
|--------|--------|----------|------------|
| 뤼이드 산타 | ✅ IRT 기반 | ❌ 고정 문제은행 | ❌ 유한 |
| 콴다 | ⚠️ 단순 추천 | ❌ 고정 | ❌ 유한 |
| ChatGPT/Claude | ❌ 없음 | ⚠️ 범용 생성 (QC 없음) | ⚠️ 무한이나 품질 보장 없음 |
| **풀림 무한풀기** | **✅ IRT + AI** | **✅ CUREA DEEP** | **✅ 무한 + 품질 보장** |

#### 플라이휠
학생이 문제를 풀수록 → IRT 파라미터 정밀화 → 난이도 메타데이터 재보정 → 적응형 추천 정확도↑ → 학습 효과 향상 → 더 많은 학생 유입

#### 확장성
IRT + 실시간 생성 아키텍처는 과목·학년에 무관하게 작동. TOEIC, 공무원 시험, 코딩 테스트, 의사 국시 등 동일 적용 가능.

### 4.2 풀림 인덱스 (Body Scan) ★★★

> *"내 학습 체력을 정밀 측정하고, AI가 맞춤 처방을 내려준다."*

기출 기반 진단평가로 학생의 학습 상태를 과목·단원·사고유형별로 정밀 측정한 후, 스토어·스튜디오 콘텐츠에서 취약점에 맞는 학습 자료를 자동 처방하는 진단-처방 일체형 시스템.

#### 학생이 경험하는 흐름
```
[진단] 과목 선택 → 15~25문항 기출 기반 레벨테스트
  → [결과] 인덱스 리포트: 단원별 히트맵 + 사고유형별 레이더차트 + 예상 등급
  → [처방] "당신에게 필요한 학습 경로":
         ① 취약 단원 → 풀림 무한풀기로 집중 훈련
         ② 추천 교재 → 스토어에서 바로 구매
         ③ 개념 보충 → 풀림 비주얼로 시각화 학습
  → [추적] 1주/1개월 후 재진단 → 성장 비교 리포트
```

#### 기술적 해자

| 해자 요소 | 경쟁사 복제 난이도 |
|----------|-----------------|
| 기출 문항 IRT 사전 계산 | 높음 |
| 3-Depth 취약점 매핑 (교육과정 분류 체계) | **매우 높음** |
| **3축 연결 처방** (스토어 교재 + 스튜디오 문제 생성 + 스터디 학습 경로) | **매우 높음** |
| 처방 효과 추적 (학습 후 해당 유형 정답률 변화 측정) | 높음 |

#### 3축 연동 효과
- 진단 결과 → 스토어 "이 교재가 당신에게 필요합니다" 추천
- 추천 교재 학습 후 → 성적 변화 측정 → 교재 효과 수치화 (스토어 리뷰)
- 콘텐츠 부족 영역 → 스튜디오에 "이 유형의 콘텐츠 수요가 높습니다" 신호

### 4.3 풀림 오답정복 (Conqueror) ★★★

> *"틀린 문제를 모으는 게 아니라, 틀리는 패턴을 정복한다."*

AI가 오답의 근본 원인을 **패턴 수준**에서 분석하고, CUREA DEEP으로 해당 패턴을 정복하기 위한 맞춤 문제 세트를 자동 생성하는 능동적 오답 정복 시스템.

#### 기존 오답노트와의 차이

| 비교 항목 | 기존 오답노트 | 풀림 오답정복 |
|----------|-------------|-------------|
| 수집 방식 | 학생이 직접 등록 | 풀림 생태계 전체에서 자동 수집 |
| 분석 수준 | 틀린 문제 목록 | **패턴 분석** (어떤 유형에서 왜 틀리는지) |
| 대응 방법 | 같은 문제 다시 보기 | **패턴 맞춤 유사 문제 자동 생성** |
| 정복 확인 | 없음 | 3회 연속 정답 → 정복 스탬프 |
| 복습 연동 | 없음 | 기억장치 자동 연동 (망각 곡선) |

#### 패턴 분석 출력 예시
```json
{
    "pattern_id": "ENG_BLANK_LOGIC_001",
    "description": "영어 빈칸 추론에서 접속사 뒤 논리관계 파악 미흡",
    "frequency": 7,
    "root_cause": "however/therefore 등 전환 접속사 후 주장 방향 판단 오류",
    "conquest_plan": { "total_questions": 5, "difficulty_range": [0.5, 0.7] }
}
```

### 4.4 풀림 튜터 (Socrates) ★★★

> *"정답을 알려주지 않는다. 질문으로 스스로 답을 찾게 해준다."*

교육과정 분류 체계를 기반으로 문제의 출제 의도·핵심 개념을 파악한 상태에서 단계적 질문을 생성하여, 학생이 스스로 답을 찾도록 유도하는 **5단계 AI 대화형 튜터**.

#### 5단계 힌트 시스템

| Level | 전략 | 예시 |
|-------|------|------|
| 1 | 방향 제시 | "어떤 부분을 다시 읽어볼까?" |
| 2 | 핵심 개념 환기 | "이 문제는 인과관계를 묻고 있어" |
| 3 | 구체적 단서 | "3번째 문장의 'because'에 주목해봐" |
| 4 | 거의 정답 | "A와 B 중에서, because 절과 일치하는 것은?" |
| 5 | 해설 공개 | 학생이 원할 때만 공개 |

#### ChatGPT/Claude와의 차이
범용 LLM은 문제를 주면 해설을 생성할 수 있지만, **그 문제가 교육과정에서 어떤 성취 기준에 해당하는지, 출제자가 어떤 사고를 요구하는지, 어떤 오답 유인이 설계되어 있는지를 모른다**. 풀림 튜터는 CUREA DEEP의 문제 분석 데이터를 기반으로 "교육적으로 의미 있는 질문"을 생성한다.

### 4.5 풀림 기억장치 (Memory Lane) ★★

> *"내가 공부한 모든 것을 AI가 기억하고, 알아서 다시 꺼내준다."*

풀림 생태계 전체에서 학습한 모든 내용을 AI가 기억하고, **개인별 망각 패턴**을 학습하여 최적의 시점에 최적의 형태로 복습을 제안하는 장기 학습 기억 시스템.

#### 전환 비용(Switching Cost) 극대화
학생이 풀림을 6개월 사용하면 → "나만의 망각 곡선", "취약한 개념 패턴", "정복한 유형 히스토리"가 모두 풀림에 축적됨 → 다른 서비스로 이동하면 이 모든 기억이 사라지고 처음부터 시작해야 함.

#### 핵심 기술
- 에빙하우스 망각 곡선 + 개인별 파라미터 베이지안 추정
- 크로스앱 이벤트 버스 (무한풀기+오답정복+모의고사+스토어 교재 학습을 하나의 타임라인에 통합)
- 복습 형태 다양화 (플래시카드, 빈칸, 유사 문제 재출제)

## 5. Growth 기능 — 성장 견인 (6개)

### 5.1 풀림 모의고사 (Exam Arena)

> *"AI가 나만의 약점을 조합한 맞춤 모의고사를 출제한다."*

실제 시험 환경을 완벽히 재현하면서, 인덱스 진단 + 오답정복 데이터를 조합하여 **"나의 취약점만 모은 모의고사"**를 AI가 자동 생성하고 성적을 예측한다. 3축 통합 없이 구현 불가능.

### 5.2 풀림 코치 (Agent Orchestra)

> *"6명의 AI 전문가가 내 학습을 코칭한다."*

6개 전문 에이전트(플래닝·튜터·모티베이션·분석·큐레이션·코치)가 협업하는 오케스트라 시스템. Core 기능 데이터가 충분히 쌓여야 각 에이전트가 정확한 판단을 내릴 수 있어 Growth 단계에 배치.

| 에이전트 | 역할 | 학생이 보는 모습 |
|---------|------|----------------|
| **플래닝** | 주간/월간 학습 계획 자동 수립·조정 | "이번 주 학습 플랜이에요" |
| **튜터** | 문제 풀이 가이드 (= 풀림 튜터) | "이 문제 같이 풀어볼까?" |
| **모티베이션** | 동기 부여·이탈 방지 | "3일 연속 학습 대단해!" |
| **분석** | 성적 예측·취약점 진단 | "이 유형 정답률이 올라가고 있어" |
| **큐레이션** | 스토어·스튜디오 맞춤 콘텐츠 추천 | "이 교재가 딱 약점에 맞아" |
| **코치 (Orchestrator)** | 전체 조율, 일관된 코칭 제공 | 학생에게 하나의 목소리로 통합 전달 |

### 5.3 풀림 풀이분석 (Process X-Ray)

> *"정답인지가 아니라, 어떻게 풀었는지를 분석한다."*

풀이 과정 자체를 데이터화: 문제별 소요 시간 분포, 선택지 변경 이력, 밑줄/하이라이트 위치, 스크롤 패턴 → 찍기 탐지, 불확실성 맵, 읽기 전략 진단, **메타인지 리포트** 생성. 한국 에듀테크에서 아무도 시도하지 않은 혁신 영역.

### 5.5 풀림 클래스봇 (ClassBot) ★★★ — 신규

> *"교사가 만들고 통제하는 AI 학습 교실. 학생은 배우고, 교사는 모든 것을 본다."*

교사/강사가 과목·주제·범위·보안수준을 설정하여 맞춤형 AI 학습 챗봇을 생성하고, 학생들이 해당 챗봇 안에서 과제·리서치·문제풀이·토론 등을 수행하는 **교사 통제형 AI 학습 환경**. B2B/B2G 매출의 핵심 드라이버.

#### 왜 이 기능이 필요한가

범용 AI(ChatGPT, Claude)를 교실에서 사용하면 다음 문제가 발생:

| 범용 AI의 교실 사용 문제 | 풀림 클래스봇의 해결 |
|----------------------|-------------------|
| 학생이 수업과 무관한 질문 가능 | 교사가 주제·범위를 사전 설정, 벗어나면 차단 |
| 교사가 학생 활동을 모니터링할 수 없음 | 실시간 대시보드 + 이탈 감지 알림 |
| 학생이 AI에게 답을 그대로 베껴씀 | "사고 유도 모드"로 답 대신 힌트 제공 (튜터 연동) |
| 학습 효과 측정 불가 | AI가 상호작용을 분석하여 학생별 점수·리포트 자동 생성 |
| 유해·부적절 콘텐츠 노출 위험 | 교육 특화 안전 필터 + 기관 맞춤 보안 정책 |

#### 4대 서브 모듈

**A. 클래스봇 빌더**: 과목·주제·학습목표·기간·대상학년·참고자료업로드·과제설정·AI행동모드를 노코드로 설정

**B. 보안 가디언 (5단계 개방도)**:

| Level | 이름 | 설명 | 적합한 상황 |
|-------|------|------|-----------|
| 1 | 수업 한정 | 교사가 업로드한 자료·설정한 주제 내에서만 AI 응답 | 시험 대비, 특정 단원 집중 |
| 2 | 주제 제한 | 설정된 과목·단원 범위 내 자유 대화 허용 | 일반 수업 보충 |
| 3 | 교과 범위 | 설정된 과목 전체 범위 자유 대화 | 자기주도 학습 시간 |
| 4 | 교육 범위 | 모든 교과·학습 관련 주제 허용, 비교육적 질문만 차단 | 수업 리서치, 프로젝트 |
| 5 | 완전 개방 | 모든 주제 허용 (유해 콘텐츠 필터는 항상 작동) | 창의적 탐구, 진로 탐색 |

**C. 실시간 모니터링 대시보드**:
- 전체 현황: 참여 학생 수, 활성/저활동/이탈감지 현황
- 학생별 상태: 질문 횟수, 현재 활동, 마지막 활동 시간
- AI 이탈 감지: 주제 이탈, 답 베끼기 시도, 부적절 질문, 장시간 무활동, 반복 이탈, 우회 시도 → 자동 경고·차단·교사 알림
- 교사 개입: 직접 메시지 삽입, 전체 공지, 개별 세션 제어, 실시간 대화 열람

**D. AI 학습 분석 리포트**:
- 학생별 평가: 참여도(20%) + 사고 깊이(30%) + 과제 달성도(30%) + 자기주도성(10%) + 팀원 기여도(10%) 자동 채점
- 리포트 6종: 실시간 대시보드, 수업 종료 리포트, 학생 개인 리포트, 기간별 추이, 학급 비교, 학부모 리포트

#### AI 응답 모드

### 5.6 풀림 플래너 (Planner) — 신규

> *"시험 일정만 입력하면, AI가 분 단위로 완벽한 학습 계획을 짠다."*

학생이 시험 일정·과목·범위·수행평가 여부 등을 입력하면, AI가 학생의 현재 실력(인덱스 데이터)과 가용 시간을 분석하여 **시간 블록(Time-Block)** 수준의 학습 일정을 자동 생성하고, 각 시간 블록에 실제 활용할 문제·개념 자료를 함께 배치하는 **AI 학습 스케줄러**.

#### 학생이 경험하는 흐름

```
[입력] 시험 정보 입력 위저드
  → 시험 종류·일시·과목·범위·수행평가·하루 가용 학습 시간·컨디션·선호
[진단] AI 진단 단계
  → 풀림 인덱스 기존 데이터 호출 → 과목별 능력치(θ) 확인
  → 미진단 과목 → 5분 퀵 진단(10문항 미니 인덱스) 자동 실행
  → 과목별 "목표 달성 거리" 산출
[생성] AI 학습 플랜 자동 생성
  → D-day까지 일별 → 시간 블록별 학습 계획
  → 과목 간 최적 배분 (약한 과목에 더 많은 시간)
  → 인지 부하 관리 (어려운 과목은 집중력 높은 시간대에)
  → 수행평가 마감일 역산 → 준비 시간 자동 확보
[자료] 시간 블록별 학습 자료 자동 배정
  → 개념 학습 블록 → 풀림 비주얼 VLM + 개념 카드
  → 문제 풀이 블록 → 풀림 무한풀기 (범위·난이도 사전 설정)
  → 취약점 보강 블록 → 풀림 오답정복 정복 세트
  → 암기 블록 → 풀림 기억장치 스페이스드 리피티션 세션
  → 모의 시험 블록 → 풀림 모의고사 범위 지정 출제
  → 개념 질문 → 풀림 튜터 바로 연결
[실행] 학습 실행 & 실시간 추적
  → 플랜 시작 버튼 → 해당 시간의 학습 자료 바로 실행
  → 타이머 + 집중도 측정 → 완료 시 자동 체크 + 성취도 기록
[적응] AI 실시간 플랜 조정
  → 오늘 계획 미달성 → 내일 자동 재배치
  → 특정 과목 정답률 급상승 → 해당 과목 시간 축소, 약한 과목에 재배분
  → "오늘 수학 더 하고 싶어요" → 자동 조정 + 나머지 과목 AI 자동 리셔플링
```

#### 5대 서브 모듈

1. **시험 입력 위저드**: 교육과정 3-Depth 자동완성, 학교별 범위 템플릿 공유
2. **진단 연동 엔진**: 인덱스 데이터 호출 → 과목별 "목표 도달 거리" 산출
3. **스케줄 생성 엔진**: Constraint Satisfaction + 인지 부하 모델로 최적 배치
4. **자료 배정 엔진**: TimeBlock × 학습목표 → 풀림 기능 자동 호출
5. **적응형 리밸런서**: 매일 학습 결과 → 잔여 플랜 자동 재최적화

#### Core 기능과의 연동

| 연동 기능 | 연동 방식 |
|----------|----------|
| 풀림 인덱스 | 과목별 능력치(θ) → 시간 배분 기준 |
| 풀림 무한풀기 | 문제 풀이 블록에 범위·난이도 자동 설정 |
| 풀림 오답정복 | 취약점 보강 블록에 정복 세트 자동 연결 |
| 풀림 튜터 | 개념 질문 시 즉시 연결 |
| 풀림 기억장치 | 암기 블록에 스페이스드 리피티션 자동 트리거 |
| 풀림 모의고사 | 주말 모의고사 블록에 범위 지정 시험 생성 |
| 풀림 비주얼 | 개념 학습 블록에 VLM 자동 배정 |
| 풀림 코치 | 코치의 플래닝 에이전트가 플래너의 핵심 엔진으로 작동 |

## 6. Future 기능 — 비전 (3개)

### 6.1 풀림 스터디룸 (Study Room)
> *"AI가 설계하는 그룹 스터디."*

개인별 학습 데이터 기반으로 그룹 활동을 자동 설계하는 소셜 학습 플랫폼. Core 기능 데이터가 충분히 축적된 후에만 의미 있는 그룹 매칭이 가능.

### 6.2 풀림 보이스 (Voice Mode) — 신규
> *"듣고 말하면서 공부한다."*

음성 튜터, 개념 팟캐스트 자동 생성, 듣기 실전 훈련, 발음 교정 등 음성 기반 학습 인터페이스. TTS/STT 기술 성숙도가 교육용으로 충분해지는 시점(2027년~)에 출시.

### 6.3 풀림 수학랩 (Math Lab) — 신규
> *"손으로 쓴 풀이를 AI가 읽고 피드백한다."*

태블릿/종이 풀이 과정을 AI가 인식·분석하여 단계별 논리 검증 + 피드백 제공. 필기 인식 + 수학적 논리 검증의 결합은 기술 성숙도가 높아야 함.

## 12. 아이데이션 미팅 주요 논의사항 (2026-03-26)

### 풀림 스터디 기능에 대해
- 새 기능(스터디)은 세부 기능 명세 없이 **아이디어 - 프로토타입 레벨**에서 우선 논의

- 유형 분류 등 콘텐츠 생성에 필요한 기반이 마련된 후 다양한 기능 시험

- qgen에 대한 별도의 docs/ 유지관리를 통해 컨센서스 싱크
  - 기능 분류 기준: cost(공통), quality-quantitative(개발), quality-qualitative(기획), performance(개발), total(공통 의사결정)

### 마케팅
- 랜딩 페이지·기능 소개 등 마케팅 자산이 기획 단계에서부터 잡혀있어야 함 (대표님)
