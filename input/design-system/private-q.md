- 해설은 사실 "이해 → 재시도 → 강화" 3페이즈로 묶인다. 12 섹션은 분해된 결과지 학생 멘탈 모델 아님.
- 4xx 404 라우트는 즉시 픽스 또는 가드.

### 개선안 — 12 섹션 → 3 페이즈 묶음

```
PHASE 1 · 이해하기            ← 펼침 default
├ Hero Recap    핵심 한눈에     ← 헤더 카드 (gradient brand)

├ Error Anatomy 오답의 해부
├ Step Replay   풀이 단계
└ Try Again     비슷한 1문항

PHASE 3 · 더 단단히            ← 접힘 default
├ Concept Deep  개념 보강
├ Similar Set   유사문항 3
├ Common Traps  자주 틀리는 길
├ Drill Pack    오늘의 드릴
```

규약:
- **각 페이즈 H2 = 20/600**, 페이즈 안 섹션 H3 = 18/600. 위계 회복.
- 모든 영문 라벨은 **"영문 · 한글" 페어 표기** (예: `Hero Recap · 핵심 한눈에`). 영문 작게(12px), 한글 18px.

- 페이즈 사이 `4px brand.100` 구분선 + 페이즈 번호 큰 숫자(48px ghost) 좌측.

### 우선순위
**high · M** — 카피 + 그룹화 + 접힘만 추가하면 즉시 인지부하 개선.

## 5. AI 코치 (Talk)

### 현황

- FAB "AI에게 묻기"가 다른 모든 페이지에서 동일 위치 (talk 진입점).
- 1차 감사: 모바일에서 FAB가 탭바·채팅 입력창과 겹침.

### 문제·기회 (Classbot 대비)
- **Classbot**은 페르소나(수학이 형/영어 누나) 중심 — 친구·강사 메타포.
- **Q의 Talk**는 *문항 컨텍스트* 중심이어야 함 — "내가 방금 푼 문항에 대해" 묻는 도구.
- 그러나 현 UI는 일반 챗봇과 구별이 안 됨. **컨텍스트 칩**(현재 보고 있는 문항/단원)이 채팅 상단 sticky로 없음.

### 개선안

- **상단 컨텍스트 카드** (sticky, 56h): "지금 보고 있는: 수학·수열·24번" + ✕로 컨텍스트 해제. 컨텍스트 있는 동안 봇 응답은 그 문항에 한정.
- **빠른칩 동적화**: 일반("힌트 하나만 줘") + 컨텍스트("이 풀이의 2단계가 왜?") 혼합.
- **수식 입력**: `$$ … $$` 단축 또는 "수식" 토글 → KaTeX 미리보기.
- **봇 응답에 코드/수식 블록 시 한 줄 액션 칩**: `이 풀이로 다시 풀어보기 →`.
- **FAB 위치**: `bottom: calc(64 + 16 + 56)` (탭바 + 입력창 + 마진). 채팅 화면에서는 FAB 숨김.

### 우선순위

## 7. 홈 7-섹션 종형 IA

### 현황
- `/q` 홈: 1차 감사에서 가장 호평한 영역. "오늘의 Q → 풀림에서 더 → 지금 → 오늘 풀이 큐 → 이번 주 → 다가오는 → 더 깊이" 7-섹션.
- 각 섹션 시그니처 `H1 — 한 줄 부제` 패턴이 의도를 즉시 전달.
- 캡처: `desktop-home-fab-open-fold.png` 외.

### 문제·기회
- 7섹션은 학습 동기 흐름(긴급 → 일상 → 확장)을 잘 따라가지만, **각 섹션의 우선순위가 사용자 컨텍스트에 무관하게 고정**됨.
- 빈 상태(긴급 오답 0건, 곧 다가오는 시험 없음) 처리 화면 미확인 → 빈 카드가 통째로 노출될 위험.
- 7섹션이 모두 같은 카드 스타일이라 "지금"(긴급)이 시각적으로 묻힘.

### 개선안

- **"지금" 섹션만 시각 차등**: warning gradient hero(`from #FFF7E6 to #FFEED1`, `1px solid warning.border`, 좌측 5px `warning.cta-bg` 막대). 다른 섹션은 white.
- **섹션 자동 숨김**: bucket 비면 섹션 자체를 숨기고 아래 섹션 끌어올림 (빈 카드 회피).
- **재정렬 옵션**: 헤더 옆 점3개 → "내 우선순위로 재정렬". 학생이 "다가오는 시험"을 먼저 보고 싶을 수 있음.
- **부제 시그니처를 토큰화**: `Section.subtitle` 컴포넌트로 — 12px `text.tertiary`, lh 16, `font-weight: 500`. 영문 em-dash는 모바일에서 `<br>` + ' — '으로 보정.

### 우선순위

- **풀문장 H2를 display 사이즈로 승격**: 28px / 700 / lh 36. 한 화면 한 문장씩 스크롤 = 가치 1개당 1뷰포트.
- **5 → 3 블록 축약**: ① 12-섹션 깊이 / ② 오답 자동 관리 / ③ AI 코치 옆에. "성장 그래프"는 ②에 흡수.
- **"준비됐어요" 다음에 단일 primary CTA 1개만**: `[지금 한 문항 풀기 →]`. 회원가입은 그 다음 화면에서.
- 모션: 각 블록 진입 시 fade-up 24→0px (M5 참조), 첫 블록만 키 비주얼 8s loop 마이크로 모션.

### 우선순위
**low · M** — 전환율에는 큰 영향, 그러나 핵심 사용 흐름은 아님.
