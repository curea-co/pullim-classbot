## 0. 한 줄 결론

> **Classbot은 "AI 분신 챗봇" 컨셉을 시각 언어로 거의 표현하지 못하고 있다.** 봇 5종이 이름·이모지만 다르고 톤·색·모션·말풍선이 동일하다. 봇 메타 카드가 대화창의 47%(모바일)를 잡아먹어 메시지 영역이 종속변수가 됐고, 키보드를 열면 입력창 자체가 사라진다. **사람화(persona-ization)·대화 우선(chat-first)·시그니처 모션** 세 축으로 재설계해야 한다.

## 1. 봇 페르소나 시스템

### 1.1 현황
| 항목 | 관찰 |
|---|---|
| 봇 5종 | 수학이 형(L3), 영어 누나(L4), 과학 쌤(L3), 국어 누나(L4), 사회 코치(L3) |
| 시각 차별화 | 컬러 원 + 이모지(`🧑‍🏫 👩‍🏫 🧑‍🔬 👩‍💼 🧑‍🎓`)만. **배경원 모두 brand.600 단색** |
| 톤 라벨 | "친근 톤 · T2" 칩이 헤더에 있으나 봇별 차이 시각화 없음 |
| LIVE 상태 | 빨간 `*LIVE` 뱃지 + 카드 외곽선만 분홍 틴트(`#FFE9EA`)로 변함 — 좋음 |
| 첫 응답 | "서연 안녕! 수학이 형이야 🙌 …" — 호명 + 톤은 살아있음 |

### 1.2 문제 / 기회
1. **봇이 다섯이지만 시각적으로 같다** — 이모지 한 글자가 유일한 차이. 채팅 화면을 캡처해서 음소거하면 어떤 봇과 대화 중인지 알 수 없음.
2. **"친근 톤 · T2" T2가 뭔지 모름** (내부 jargon)
3. 봇 메타 카드(아바타 + 학원명 + 이름 + 칩 4개 + 알림 줄 + L3 범위 줄)가 **모바일 388px 높이** — 첫 메시지 1개 보이면 끝.
4. **봇 선택 칩(상단)**: 5개 가로 스크롤. 모바일 첫 화면에 4번째부터 잘려 보임. 활성 상태 표시(파란 배경)만 다르고, 어떤 봇이 LIVE인지 칩에서 보이지 않음 (홈 카드엔 LIVE 뱃지가 있는데 채팅 칩엔 없음 — 일관성 결함).

### 1.3 개선안

**A. 봇별 시그니처 컬러 + 모티프**

청색 브랜드 코어를 유지하되 봇마다 **액센트 1색 + 패턴 1종**.

| 봇 | 액센트 | 아바타 배경 | 모티프 | 어조 |
|---|---|---|---|---|
| 수학이 형 | `brand.600` + `lime` | navy radial | √ x² ÷ 수식 도트 | 단정·반말 |
| 영어 누나 | coral `#FF7A6B` | warm-rose | A B C 살짝 회전 | 상냥·존대 |
| 과학 쌤 | mint `#22C5A8` | sage radial | 분자/원자 도트 | 호기심·반말 |
| 국어 누나 | violet `#7B5CFF` | lavender | 한글 자모 ㅎㄱㅁ | 차분·존대 |
| 사회 코치 | amber `#D97706` | cream | 지도 라인 | 격려·반말 |

— **브랜드는 청색 사이드바·CTA로 유지**, 봇 카드·말풍선 액센트만 봇 컬러.

**B. 봇 메타 카드를 "접힌 헤더"로**

```
[ 기본 (스크롤 안함) - 56px ]
┌────────────────────────────────────────────────┐
│ ← [🧑‍🏫] 수학이 형 · L3 교과범위        ⓘ ⌃   │
└────────────────────────────────────────────────┘

[ 펼침 (탭/⌃ 시) - 180px ]
+ 학원명 / 톤(친근체) / 응답시간(~3초) / 자동차단 안내
+ 범위 변경 진입
```

**C. 봇 스위처를 "탭+드로어" 하이브리드로**

```
┌─────────────────────────────────────┐
│ [수학이 형 · LIVE]  [영어 누나]  …  │  ← horizontal scroll 5개
│                                  [≡] │  ← drawer: 봇 검색·즐겨찾기
└─────────────────────────────────────┘
```
- 현재 LIVE인 봇은 빨간 펄스 점(2px 라이너), 대기는 회색 점.
- 활성 봇은 **봇 시그니처 컬러 배경 + 흰 글자** (지금처럼 brand 단일색이 아님)

**D. 카피 정리**
- "L3 교과범위" / "친근 톤 · T2" — 외부 노출시 첫 등장에 **툴팁** 또는 한글 병기: "안전 등급 L3 (개념 설명만)", "친근 톤"으로 한정. **T1/T2/T3** 같은 내부 코드 노출 금지.

## 2. 메시지 / 채팅 화면

### 2.1 현황
- 봇 버블: 흰 배경 + 옅은 라인(거의 안 보임). 봇 이름+아바타가 버블 위에 별도로 노출 → 세로 공간 낭비.
- 사용자 버블: **관찰 안됨** (대화 데이터 없음) — 첫 응답 이후 빈 영역.
- 시간/날짜 그루핑: 없음. 메시지 위에 시간만 점·인라인.
- 메시지 종류: 텍스트 1종만 관찰. 문제·이미지·카드형 없음.
- 빠른 칩 4개: `극값 어떻게 찾아요? / 오늘 수업 요약해줘 / 4월 학평 대비 뭐 해야 해요? / 저 잘하고 있는 거예요?` — **하드코딩 추정**, 봇·과목별 동적이지 않음.

### 2.2 문제 / 기회
1. **봇 이름+아바타가 매 메시지마다 위로 튀어나옴** — 연속 발화 그루핑 패턴 부재. 채팅 도메인 표준(iMessage/Slack/ChatGPT) 미준수.
2. **봇 vs 유저 시각 구분 약함** — 봇 흰 배경 + 옅은 라인, 유저 버블 미관찰. 추정컨대 brand.50 tint일 텐데 두 톤 다 옅어 스크롤 시 누가 누구인지 빠르게 안 잡힘.
3. **메시지 타입이 텍스트 한 종**. 문제 풀기·해설·이미지·웹링크 카드·진도 카드 등 챗봇 핵심 UX가 비어있음.
4. **시간/날짜 디바이더 없음** — 어제 대화/오늘 대화 연속 표시 시 혼란.
5. **첫 응답 카피 좋음** ("서연 안녕!") — 호명·이모지·자기소개 OK. 다만 늘 같은 문장일 가능성 (랜덤화 필요).

### 2.3 개선안

**A. 버블 시스템 (3 variant)**

```
[Bot bubble - 첫 발화]
┌──────────────┐
│🧑‍🏫 수학이 형  · 19:50
│┌─────────────────────────┐
││서연 안녕! 수학이 형이야 🙌
││오늘 미적분 III장 진행 중…
│└─────────────────────────┘  ← bg #FFFFFF, border #EDF0F5
└──────────────┘

[Bot bubble - 연속 발화 (3분 이내 같은 봇)]
   ┌─────────────────────────┐
   │극값 판정은 보통 2계 도함수…
   └─────────────────────────┘  ← 아바타 생략, 32px 들여쓰기

[User bubble]
                    ┌─────────────────────────┐
                    │적분 어떻게 시작해?      │  ← bg brand.600, white text
                    └─────────────────────────┘
                                         19:52 ✓
```

- **봇 액센트 라이너**: 첫 버블 좌측 3px 라인 = 봇 시그니처 컬러
- **유저 버블은 채워진 brand.600** (지금처럼 옅은 brand.50보다 명확)
- **체크마크**: 전송/수신/AI생성 상태 (✓/✓✓/⚡)

**B. 메시지 타입 6종 카탈로그**

| 타입 | 비주얼 단서 | 용도 |
|---|---|---|
| text | 기본 버블 | 일반 답변 |
| problem-card | 라임 좌측 라이너, 문제번호 큰 숫자, "풀러 가기" CTA | 과제·퀴즈 인라인 |
| explain-step | 1️⃣2️⃣3️⃣ 단계 indent, 수식 mono | 단계별 풀이 |
| reference-link | 16:9 썸네일 + 도메인 + 제목 + 한 줄 요약 | 외부 자료 |
| image | 라운드 14, 탭하면 lightbox | 사진/필기 |
| audio (T3) | 파형 + 재생/배속 | 음성 답변 (영어 누나 발음 등) |

**C. 시간/날짜 그루핑**

```
─────────── 오늘, 5월 19일 ───────────
[메시지들]
─────────── 어제 ───────────
[메시지들]
```
12px / `text.tertiary` / 32px 상하 여백. iMessage 패턴 그대로.

**D. 빠른 칩(Quick Reply) 동적화**

현재 4개 하드코딩 → **컨텍스트 기반 동적 3종**:
1. **과목 입문 질문** (대화 시작 시): "오늘 진도 알려줘" / "지난 시험 분석해줘"
2. **마지막 봇 발화 기반 후속 질문**: 봇이 "극값" 설명 → 칩 "극값 예제 더 보여줘"
3. **시간/상황 기반**: 새벽엔 "오늘 컨디션 안 좋아" 같은 웰빙 진입 칩

칩 표시 위치는 **인풋 위에 stack 2줄**, 좌측 시그니처 컬러 라이너로 봇 추천임을 시각화.

**E. 첫 응답 personalization**
- 호명("서연")은 OK. 다만 **시간대 인사** ("좋은 아침이야 서연" / "수업 끝났네! 고생했어")
- **봇별 시그니처 인사** 등록 (수학이 형: "오늘 미적분 어디까지 갔어?" / 영어 누나: "오늘은 어떤 지문 같이 볼까요?")

## 3. 입력창

### 3.1 현황
| 측정 | 값 |
|---|---|
| 위치 (데스크탑) | y=760, w=1066, h=39 — 화면 하단 fixed |
| placeholder | "수학이 형에게 물어보세요…" (봇명 치환 OK) |
| multiline | 미관찰 (높이 39px 고정 — single line) |
| 첨부 진입점 | **없음** |
| 전송 버튼 | 우측 원형 brand.600 종이비행기 아이콘 |
| FAB | 메인 화면에는 안 보임 (좋음) |
| 모바일 키보드 열림 | **입력창 자체가 화면에서 사라짐** (M-keyboard-open.png 참고) — Critical 버그 |

### 3.2 문제 / 기회
1. **모바일 키보드 = 입력창 실종**: viewport 460px로 줄이는 시뮬레이션에서 입력창과 빠른칩이 모두 사라지고, 봇 헤더 카드와 첫 메시지만 보임. 실제 iOS visual viewport 안 잡힌 듯.
2. **첨부 진입점 0개**: 사진 찍어서 풀이 질문, 음성 녹음(영어 발음), 필기 노트 업로드 등 학생 챗봇 핵심 입력 채널 없음.
3. **single-line 추정**: 긴 질문 작성 어려움. multi-line 자동 확장 필요.
4. **전송 버튼 옆 영역**: 빠른칩이 위 한 줄, 인풋이 아래 한 줄, FAB 없음. 좋은 분리. 단, **음성 입력 마이크** 자리 없음.
5. **Scope L3 / Scope L4 — 모든 입력 영역 풋터에 노출** ("Scope L3(교과 범위) — 같은 학년 학과 위주 1탬담"). 학생 대상 UI에 내부 jargon 노출.

### 3.3 개선안

**A. 입력바 재설계**

```
┌────────────────────────────────────────────────────────┐
│ [+] [🎤] [수학이 형에게 물어보세요…             ]  [→]│
└────────────────────────────────────────────────────────┘
   ↑     ↑    ↑                                          ↑
첨부메뉴 음성 multiline 자동확장 (max 5 lines)        전송
```

- `[+]` 탭 → **drawer 위로**: 사진 / 카메라 / 필기 / 문제 캡처 / 음성 메모
- `[🎤]` 길게 누름 → 음성 입력 (영어 누나 발음 입력에 필수)
- placeholder: 봇 이름 + **시그니처 prompt** 예시 ("수학이 형에게 — 막힌 문제 사진 찍어 봐도 돼")

**B. 모바일 키보드 대응 (Critical)**
- `visualViewport` API로 키보드 높이 측정, **입력바 + 빠른칩**을 키보드 위로 sticky.
- 키보드 열릴 때 메시지 영역은 자동 스크롤 (마지막 메시지 → 입력창 위로).
- 봇 메타 카드는 **자동 collapse** (탭하면 펼침).

**C. Scope 표시 자리 이동**
- 입력바 아래 풋터 → **봇 헤더 카드 내부**(접힘 상태에서만 짧게 "L3"). 매 키 입력마다 보이는 자리에서 빠짐.

## 4. LIVE 수업 & 리플레이

### 4.1 현황
| 영역 | 관찰 |
|---|---|
| LIVE 카드 (홈) | 다크 그라데이션 navy + 라임 progress + 라임 CTA pill |
| LIVE 뱃지 | 빨간 `*LIVE` pill (`#E5484D` bg, white text) |
| 리플레이 카드 | 큰 다크 카드 2종 — (a) 이어 보기 카드 (50% 시청), (b) 방금 끝난 수업 카드 |
| 진행 막대 | 라임 50% — 좋음 (lime이 진척률 신호) |
| 시청자 수 | "14명 참여 중" 작게 표기 |
| CTA | "25:00부터 이어서 →" (라임 pill + 검정 텍스트) — 명확 |
| 시간 라벨 | "25:00 ... 50:00" 라임 라인 양끝 — 정보량 좋음 |

### 4.2 문제 / 기회
1. **다크 카드가 generic**: 그라데이션 navy → blue. 다른 사이트와 차별점 없음. 라임 + ink의 **시그니처 콤보** 활용 부족.
2. **LIVE 뱃지에 모션 없음**: 정적 빨간 pill. "지금 켜져 있다"는 신호가 약함.
3. **시청자 수 표기가 정적**: "14명 참여 중" — 실시간 증감 시각화 없음.
4. **리플레이 카드 2개가 동일 비주얼 가중치**: "방금 끝난 수업" vs "이어 보기" 우선순위 안 보임.
5. **썸네일 영역 미사용**: 리플레이는 본질적으로 영상. 16:9 thumbnail + 자막 발췌가 더 유용.

### 4.3 개선안

**A. LIVE 카드 시그니처 (solid + 라임 라이너)**

```
┌─━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┐
│●━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━│  ← 좌측 4px lime 라이너
│  ●LIVE ⬢ 14명 참여 중 · 23:14 진행            │
│  수학이 형 라이브                              │  ← 22px bold white
│  도함수의 활용 — 극값과 변곡점                 │
│                                                │

```

4. **빈 상태**: 과제 0건 일 때 화면 미관찰. "받은 과제가 없어요" 카피와 봇 추천 진입 필요.

## 6. 웰빙

### 6.1 현황
- 헤더: "오늘 어땠어요?" 24px bold + "오늘 체크인 완료 — 내일 또 와주세요"

- 주간 기분 기록: 이모지 + 날짜 + "그럭저럭/좋아/그저그래" 리스트

### 6.2 문제 / 기회
1. **봇과의 연계가 시각적으로 없음**: 웰빙은 봇이 개입하는 영역인데 봇 아바타·이름이 페이지에 한 번도 안 나옴.
2. **78점이 무엇인지 모름**: "5지표 가중 평균"이라고만 적힘. 5지표가 뭔지 펼침 없음.
3. **막대 7개가 단조롭게 같은 brand.600**: 좋은 날/나쁜 날 색 차이 없어 trend 파악 어려움.
4. **이모지가 OS 기본** (😐😊😐😞): 봇 페르소나와 분리됨. **봇 시그니처 무드 셋** 필요.
5. **체크인 액션 사후 피드백 부재**: 완료 후 봇이 어떻게 반응하는지 노출 안됨.

### 6.3 개선안

**A. 봇 개입 시각화**
```
┌─ 오늘의 봇 코멘트 ───────────────────────┐

## 7. 홈 4블록

### 7.1 현황
| 블록 | 데스크탑 | 모바일 |
|---|---|---|
| 내 클래스봇 | 5개 카드 그리드 3열·2행 | 1열 세로 — LIVE 봇 2개가 위 |
| 도함수 활용 마무리 (진행 과제) | 큰 navy 카드 + 라임 progress + lime CTA | 동일 (스크롤로 나타남) |

| 지금 라이브 퀴즈 (1/14문항 응답 79%) | 흰 카드 + 객관식 1234 | — |

### 7.2 문제 / 기회
1. **"내 클래스봇" 블록이 1순위인데 정보량 부족** — 카드에 봇명+학원+학년만. **마지막 메시지 미리보기 / 미완료 과제 수**가 빠짐. iMessage 채팅 리스트처럼 진입 후크가 약함.
2. **LIVE인 봇 2개가 위 (좋음)** — 단, 분홍 틴트가 약해 흑백 모드에서 구분 안됨.
3. **블록 순서 (데스크탑)**: 봇 → 진행 과제 → LIVE 진행 중 → 라이브 퀴즈 → 받은 과제. **LIVE 진행 중이 3번째**라 학생이 라이브 놓침. → "지금 일어나는 일" 블록을 1번째로.
4. **데스크탑 그리드 3열**: 5개 봇이 3열 2행 = 마지막 줄에 빈 칸 1개. 4열로 가거나 5열 단일 행이 시각적으로 더 깔끔.
5. **모바일 LIVE 봇 카드는 클릭 시 어디로?** — 채팅인지 라이브 입장인지 불분명. **카드 안에 2 CTA** 필요.

### 7.3 개선안

**A. 블록 우선순위 재배치 (시간 가치순)**
```
1. 지금 LIVE (있을 때만) — 적색 펄스 + "들어가기"
2. 내 클래스봇 (5개) — 메시지/과제 미리보기 포함
3. 진행 중 과제 1개 (가장 임박)
4. 오늘의 KPI / 웰빙 한 줄
5. 최근 리플레이
```

**B. 봇 카드 정보 보강**
```
┌──────────────────────────┐
│ [🧑‍🏫] 수학이 형 ●LIVE     │
│       김수학 · 고2 미적A │
│  ─────────────────────── │
│  💬 "오늘 진도 잘 따라왔어"
│  📋 미완료 1 · 라이브 1   │
│                  [입장 →] │
└──────────────────────────┘

- 미완료/라이브 카운트 (icon+숫자)
- 1차 CTA만 노출 (LIVE면 "입장", 아니면 "대화")

**C. 그리드 (데스크탑)**

- 알림(상단 종 + 빨간 점) 펼침 시 UI
