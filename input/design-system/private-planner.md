## 0. 한눈에 보기

Pullim Planner의 정체성은 **"수능까지 남은 시간을 시각·감정·진행도로 동시에 보여주는 학생용 학습 OS"** 다. 일반 캘린더 앱이라면 무난할 패턴들이 학생 수험 도메인을 만나면 다른 결정을 내려야 한다 — 예컨대 "충돌"은 회의실 충돌이 아니라 **체력·연속 집중력 한계**이고, "빈 슬롯"은 free time이 아니라 **회복 시간 vs. 보강 기회**의 갈림길이다.

8개 영역 모두에서 공통적으로 **"학생을 다그치지 않으면서도 진행을 유도하는"** 톤을 더 정교하게 만들 여지가 크다. 시각·모션·카피가 한 방향(부담 < 안도 < 자율감)으로 정렬되어야 한다.

## 1. 시간 기반 캘린더 그리드 (12–22시 세로 시간표)

### 현황
좌측 "오늘 일과" 카드. 12~22시 11행 세로 그리드(시간 라벨 9px), 가로 막대바로 블록 표시. 진행 중 블록만 진한 청색, 나머지는 옅은 청색 농도 변화. 13~14시 사이에 빨간 가로선(현재 시각). 13시 근처에 사선 빗금 패턴(미수행/지난) 막대.

### 문제·기회
- **그리드가 너무 적막**: 11시간을 동일 높이로 분할 → 학생의 핵심 시간대(저녁 17–22시)를 더 크게 줘야 한다.
- **현재 시각 라인이 "사실의 진술"에 그침**: 라벨도 없고 "지금"이라는 의미가 시각적으로 약함. 안 움직임.
- **농도가 의미를 못 담음**: 옅은 파랑 = 대기인지 완료인지 구분 안 됨. 진한 파랑 = 현재 진행 중이지만, 사선/회색/녹색이 혼재되어 색의 문법이 일관되지 않음.
- **빈 슬롯이 클릭 불가/시각적 정적**: 호버해도 affordance(예: + 추가, 회복 슬롯 추천) 없음. "여기에 뭔가 넣을 수 있다"라는 신호 0.
- **드래그 정렬 안내 문구**가 본문 우상단에 회색 텍스트로만 떠 있어 미구현임을 사용자가 책임진다(왜 안 되는데?).
- **충돌·겹침은 현재 데이터 없음** → 향후 같은 시간대 2개 블록 생기면 시각 표현 부재 위험.

### 개선안

**(a) 가변 행 높이 + 시간대 그룹**
```
점심(12-14) ── 80px 단위 그룹, "휴식 권장" 약한 배경
오후 집중(14-18) ── 120px 단위, 무지 흰 배경
저녁 코어(18-22) ── 160px 단위, 약한 강조 배경 (전체 학생 시간의 60% 차지)
```
구간 라벨을 좌측에 작게 sticky로 깔아주면 "지금이 코어 타임"이라는 메타 인식을 무료로 얻는다.

**(b) "지금" 인디케이터 강화**
- 가로선 양 끝에 작은 알약 라벨 `● 14:03 지금` (왼쪽 시간 라벨 컬럼에 살짝 돌출)
- 60초마다 부드럽게 한 픽셀씩 내려가는 micro-motion (1초 transition, 결국 시간이 흐르는 것을 보여줌)
- 라인 색은 채도 한 단계 낮추고, 그 대신 라인 좌측에 두꺼운 dot(8px)로 강조 → 빨간 가로선이 시각을 점령하지 않게.

**(c) 블록 농도 → "상태 색문법" 통일**
| 상태 | 채도/명도 | 패턴 |
|------|-----------|------|
| 완료 | 청록 #12B26B 12% 면 + 좌측 4px 진한 청록 stripe + 체크 아이콘 | 솔리드 |
| 진행 중 | primary-600 면 (한 단계만 진함) + 우측 진행률 wedge | 솔리드 + 진행 wedge |
| 대기(미래) | primary-100 면 + primary-300 border 1px | 솔리드 |
| 미수행/연체 | amber-100 면 + amber-500 사선 빗금 | 사선 |
| 회복/식사 | slate-200 면 + 🍽 아이콘 | 솔리드 무채색 |

→ "진한 파랑 = 진행 중", "사선 = 미수행", "녹색 = 완료", "회색 = 비학습"이라는 4단어 문법이 한눈에 잡힘.

**(d) 빈 슬롯 affordance**
- 호버 시 슬롯 배경 slate-50 + 점선 border + 중앙에 `+ 15분 회복 / + 25분 보강` 두 개 ghost 버튼.
- 빈 슬롯이 30분 이상이면 시스템이 회복/보강 제안 미리 채워둠 (얇은 점선 카드).

**(e) 충돌 표현 (선제 디자인)**
- 같은 시간대 2개: 50%씩 가로 분할 + 양쪽 카드 우상단에 `⚠ 충돌` 노란 dot.
- 시간대가 부분 겹침: 뒤 블록을 5px 우측 indent + 좌측 상단에 연결 곡선으로 "겹침" 표시.

**(f) 드래그/리사이즈 어포던스**
- "곧 열려요" 문구 제거. 대신 카드 hover 시 좌우 끝 8px 영역에 grip 아이콘이 fade-in (실제로 작동하지 않더라도 의도된 인터랙션 가시화). 베타 출시 전까진 카드 우상단에 `Beta ✋` 작은 칩.

**ASCII 의사 표현**
```
┌─ 오늘 일과 ───────────── D-16 · 1.9/4.4h · 1/7 ─┐
│ 점심(12-14)                                       │
│  12 ┤                                             │
│  13 ┤▓▓▓▓ 점심 직후 단어복습 (미수행, 사선)  ▶ │
```

## 2. 학습 블록 카드 시스템

### 현황
우측 "오늘의 학습 블록" 리스트 7개. 각 행 = 시각(작음) · 아이콘(원형) · 제목(16px) · 메타(어제 누락분 이월 등) · 상태뱃지(미수행/완료/진행/대기) · CTA(시작·이어서·완료) · 점3개 메뉴. 상태별 색은 잘 잡힘(amber/green/blue/slate).

### 문제·기회
- **상태머신이 카드에 직접 표현되지 않음**: 진행 중 카드가 outline 강조만 받음. "60% 진행"·"3/5 문항" 같은 진행률이 보이지 않아 "이어서"가 어디서부터인지 모호.
- **연체/미수행이 첫 행에 있으나, 시간 순서를 깬다**: 13:30 미수행 → 17:30 완료 → 18:25 진행 → 19:30~ 대기. 시간 순서 + 우선순위가 섞여서 정렬 의도가 불분명.
- **블록 타입별 시각 구분이 약함**: 아이콘(원형)이 옅은 청색 면 + 작은 lucide icon. 모의평가/단어/문제풀이/식사가 거의 같은 톤. "모의평가"는 시각적으로 더 무거워야 한다.
- **점3개 메뉴**가 모든 행에 동일. 행마다 가능한 액션이 다른데 entry는 동일 → 학습 부담.

### 개선안

**(a) 상태머신 시각화 — 카드 좌측 4px stripe + 우측 진행 wedge**
```
┌──────────────────────────────────────────────────┐
│▌13:30 ✎  점심 직후 단어 복습                  ⚠⌚│  미수행: amber stripe + ⚠ + 이월 시각 표기
│  어제 누락분 이월 · 30분                  [지금]  │
└──────────────────────────────────────────────────┘
┌──────────────────────────────────────────────────┐
│▌17:30 □  미분 기본 공식 시각화        88% ●  ✓ │  완료: green stripe + 정답률 + 감정
└──────────────────────────────────────────────────┘
┌══════════════════════════════════════════════════┐
│▐18:25 ✎  미분 — 적응형 문제 풀이      ●●●●○○ 60%│  진행: blue stripe + 진행 wedge + 강조 outline
│  D-day 빈출 가중 · 남은 12분            [이어서] │
└══════════════════════════════════════════════════┘
```

**(b) 정렬 모드 토글**
- 기본: 시간순(현재 그대로)
- 옵션: "우선순위순"(연체·D-day 가중 위가) / "코스트순"(짧은 것부터)
- 토글 위치: 헤더 "오늘의 학습 블록" 우측, 작은 sort 아이콘 + label.

**(c) 블록 타입별 아이콘 컨테이너 통일**
| 타입 | 컨테이너 | 아이콘 | 면 색 |
|------|----------|--------|-------|
| 모의평가 | 14px radius 정사각 | TrophyIcon | warning 50 |
| 문제풀이 | 14px radius 정사각 | EditIcon | primary 50 |
| 단어/암기 | 14px radius 정사각 | BookOpenIcon | violet 50 (브랜드 보조색) |
| 개념 학습 | 14px radius 정사각 | LampIcon | teal 50 |
| 식사/회복 | pill | UtensilsIcon | slate 100 |
- 모의평가는 컨테이너에 작은 D-day 칩 (예: `D-16`) 우상단에 겹쳐서 "이 블록이 그날까지 가장 중요"임을 시각화.

**(d) 점3개 메뉴 차별화**
- 미수행: [지금 시작 / 내일로 미루기 / 삭제]
- 진행: [일시 정지 / 메모 추가]
- 완료: [되돌리기 / 회고 보기]
- 대기: [지금 시작 / 시간 변경 / 노트]

**(e) 카드 hover micro-motion**
- 0.18s ease-out으로 카드가 1px 위로(translateY -1px) + shadow-sm 부여 + stripe가 1px 두꺼워짐. 클릭 어포던스.

## 3. D-day & 학사 일정 표현

### 현황
- H1 아래 메타라인: `6월 모의평가 · D-16 · 1/7 블록 완료` — 모두 같은 청색 굵은 글씨.
- 일과 카드 헤더에 `D-16 1.9h/4.4h · 1/7 블록` — D-16만 작은 알약 모양으로 강조.
- 월간 탭에 빨간 깃발 dot (6월 모의평가) — 캘린더에 잘 나타남.
- 월간 우측 패널: "큰 그림과 마일스톤" 카드, D-day 진행률 바(82%), "다가오는 마일스톤" 4월 30일·4월 학력평가.

### 문제·기회
- **D-day의 가중치 변화가 시각화되지 않음**: D-30과 D-3은 행동이 달라야 하는데 같은 톤.
- **모의평가 일정과 학습 블록 연동 신호가 약함**: `D-day 빈출 가중`이라는 작은 메타만 있고, 시각적으로 "이 블록은 그 시험 때문에 우선"임이 안 보임.
- **시험 임박 알림 디자인 부재**: D-3 이내일 때 무엇이 달라지는지 없음.

### 개선안

**(a) D-day 단계별 시각 단계 (Tier system)**
| Tier | 범위 | 시각 처리 |
|------|------|-----------|
| 평시 | D-31+ | 본문 inline 텍스트 "D-45" |
| 주의 | D-15 ~ D-30 | primary 알약, 굵게 |
| 임박 | D-7 ~ D-14 | warning 알약 + 작은 캘린더 아이콘 |
| 직전 | D-1 ~ D-6 | danger 알약 + 좌측 미세 pulse 모션 (1초 주기, opacity 0.6→1.0) |
| 당일 | D-0 | danger 솔리드 + "오늘 D-Day" 텍스트 + 헤더 띠 |

**(b) D-day 헤더 띠 (D-3 이내)**
페이지 최상단 헤더 아래에 4px 두께 띠 (warning 색). 띠 위 호버 시 "6월 모의평가까지 3일 — 컨디션 75% 이상 유지하기" 툴팁. 명시적 임박 상태.

**(c) 학습 블록 ↔ 시험 연결 칩**
블록 카드 상단에 작은 outlined 칩: `🎯 6월 모의평가 가중`. 클릭하면 동일 시험에 묶인 블록 필터링. 메타 텍스트("D-day 빈출 가중")가 시각적 칩으로 격상.

**(d) 카운트다운 미니 뷰 (헤더 우측 상시 표시)**
현재 `17일째` 스트릭 칩 옆에 `D-16 6월 모의평가` 칩. 클릭 시 마일스톤 페이지로. 어디 페이지에서든 가장 가까운 시험이 항상 시야 안.

**(e) 임박 알림의 톤**
- 위협("D-3 위험!")이 아니라 자율감("D-3 — 컨디션 안정 우선, 새 단원 No")으로. 학생을 다그치지 않는다.

## 4. 컨디션 / 번아웃 트래커

### 현황
홈에서는 접힌 상태로 한 줄: `🙂 오늘 컨디션 · 번아웃 — 보통이에요 · 64/100`. 펼치면 5단계 이모지 선택(😴😫🙂😊🤩) + 번아웃 도넛(64/100, "컨디션 관리 필요해요" warning) + 평균 수면 5.8h, 연속 블록 완료율 78%, 등 보조 메트릭. 리포트 페이지엔 컨디션 trend 7일 막대 + 번아웃 안전도 7일 막대 + "금요일 위험 신호 — 휴식 권장" amber 캡션.

### 문제·기회
- **단일 슬라이더 + 5단 이모지 두 입력의 의미 분리가 모호**: 64는 무엇의 64인가? 컨디션? 번아웃? 둘 다? 헤더에는 "컨디션·번아웃 보통이에요·64"가 한 줄로 붙어 양쪽이 같은 값처럼 보임.
- **컨디션 입력의 마찰**: 5단계 이모지 + "기본/난이도" 옵션이 있는데 학생이 매일 두 번 누를 가능성은 낮음.
- **번아웃 64는 잘 안 보이지만 "컨디션 관리 필요해요" 경고가 떠 있어 톤이 무거움**: 양쪽 메시지 톤 불일치.
- **"부모님께 보내기" 버튼이 리포트 페이지 우상단에만 있음**: 컨디션이 안 좋을 때 도와줄 사람과 연결되어야 하는데 액션 동선이 멀다.
- **시간대별 추이가 없음**: 하루 안의 변화(점심 후 떨어진다 등)를 보면 의사결정에 도움.

### 개선안

**(a) 컨디션 vs 번아웃 메타 분리**
- 컨디션 = 오늘의 감정(주관, 5단 이모지)
- 번아웃 = 연속/누적 부담(객관, 0-100)
- 헤더 한 줄을 두 칩으로 분리:
  `🙂 오늘 보통`  `🔋 번아웃 64 · 주의`
- 색 문법: 컨디션은 항상 무채색 + 이모지 색, 번아웃은 0-39 success / 40-69 amber / 70+ danger.

**(b) 한 번에 모이는 빠른 입력 (3초 입력)**
모바일 펼침 시 큰 5단 이모지가 가로로 풀폭, 한 번 탭 = 컨디션 기록 + 화면 위쪽으로 toast `🙂 오늘 컨디션 저장 — 잘 쉬어요`. 자동 fold-back 2초 후. "기본/난이도" 옵션은 작은 보조 칩으로.

**(c) 시간대별 컨디션 미니 점선**
펼침 영역 상단에 24시간 미니 라인 차트(높이 40px). 오전/점심 후/저녁 3 anchor 입력으로 충분. 회고 시점에 "점심 후 떨어졌네요. 다음 주엔 13:30 짧은 산책?" 인사이트.

**(d) 임계치 경고 UI — 차분하게**
번아웃 70+ 또는 컨디션 연속 2일 "피곤" 시:
- 메인 헤더 위 1줄 배너 (warning bg, dismissible)
- 카피: `이번 주 부담이 쌓이고 있어요. 오늘 저녁 1블록 줄여볼까요?` + [블록 줄이기] [그대로 가기] 두 ghost 버튼.
- "위험"이 아니라 "선택"으로 프레이밍.

**(e) "부모님께 보내기"의 맥락화**
- 리포트 상단 외에 **임계치 경고 배너 안에도** secondary 액션으로 노출: `[부모님께 공유]`. 도움이 필요한 순간에 도움 동선이 있다.
- 학생 톤에서 정중한 톤으로 변환: "부모님께 보내기" → "부모님께 이번 주 회고 공유". 행위가 무엇인지 분명.

**(f) 도넛 차트 톤 조정**
현재 64에서 진한 오렌지(#F59E0B) → 학생이 보기에 과한 경고. 도넛 두께를 절반으로 줄이고 배경 ring을 더 옅게. 가운데 숫자 옆에 "/100" 회색 → 압박감 완화.

## 5. 주간 성장 리포트

### 현황
KPI 4장(학습 시간 29.3h/30h, 평균 정답률 82%, 약점 정복 2건, 감정 평균 3.8/5) + 막대 차트(이번 주 학습 시간 요일별) + 라인 차트(정답률 추세) + 컨디션 trend 7일 막대 + 번아웃 안전도 7일 막대 + "약점 단원 진도" 진행 바 리스트.

### 문제·기회
- **모든 차트가 같은 청색 단조** → 변별력 약함. 컨디션 trend(이모지 + 막대)와 번아웃 안전도(녹/오렌지) 외엔 모두 청색.
- **KPI 카드 4개가 일률적**: 변화량 화살표가 작아 추세 인식이 약함. 클릭 affordance 없음(드릴다운 안 됨).
- **"한눈에 진행률 요약" 패널(주간 탭)** 보조 텍스트는 좋지만 "수학 정답률 +12% — 새 단원 진입 적기" 같은 한 줄 인사이트 카피가 더 자주 나와야.
- **회고 한 줄 카피가 적음**: 학생이 자기 진행을 글로 표현할 기회 없음.

### 개선안

**(a) KPI 카드 — 드릴다운 + 미니 sparkline**
```
┌─ 학습 시간 ────────────┐
│ 29.3h    ↑ 4h         │
│ /30h 목표             │
│ ▁▂▃▅▆▇▆ (sparkline 7일)│
│ 화 부족 → 토 보강 ✓    │ ← 자동 인사이트 한 줄
└────────────────────────┘
```
sparkline은 16px 높이로 작게, 카드 hover 시 클릭 → 해당 메트릭 상세 모달.

**(b) 차트 컬러 다양화 (브랜드 정합 유지)**
- 학습 시간 막대: primary-500/primary-200 (목표 미달 = 옅음)
- 정답률 라인: success-500
- 컨디션 trend: 이모지 색 그대로 (이모티콘 + 옅은 청색 막대)
- 번아웃 안전도: 임계치 색 (green/amber/red)
- 약점 단원 진도: 색은 진도에 따라 0-30% slate / 30-70% primary / 70%+ success

**(c) 인사이트 카드 (한 주에 3-5개 자동 생성)**
리포트 상단에 카드 슬롯:
- `📈 수학 정답률 +12% — 새 단원 진입 적기`
- `⏱ 화요일 학습 시간 부족 — 평일 시간대 점검`
- `🔋 금요일 위험 신호 — 휴식 권장`
- `💪 17일 연속 1블록 이상 완료 — 흐름 유지`

세 줄 이내 짧은 카피 + 작은 액션 칩(예: `[시간표 조정]`). 학생이 처음 보는 화면이 "데이터 덤프"가 아니라 "선생님 멘트".

**(d) 회고 한 줄 작성 UI**
KPI 영역 아래에 `이번 주 한마디` 카드 — 빈 칸 placeholder "이번 주 가장 뿌듯한 한 순간"... 자동 저장. 4주가 쌓이면 월간 리포트에 회고 인용 노출. **자기 효능감 강화**.

**(e) "부모님께 보내기"의 사전 미리보기**
버튼 클릭 시 모달에서 "공유될 내용"을 학생이 검토 + 한 줄 메시지 추가 옵션. 부모와 학생 양쪽의 통제감.

## 6. 학생 페르소나 UX 톤 (마이크로카피)

### 현황
"오늘 어땠어요?" 류 친근 톤은 거의 없음. 대신 "지금 시작", "이어서", "보통이에요", "드래그 정렬 곧 열려요" 정도. 17일 연속 스트릭은 헤더 우측 작은 칩.

### 문제·기회
- **"드래그 정렬 곧 열려요"가 가장 친근한 카피** — 다른 표면은 거의 무미건조.
- **17일 스트릭이 거의 안 보임**: 작은 칩 + 회색 톤 → 게임화 동기부여 약함.
- **카피가 "동기부여" 쪽에 치우치지 않음**: "지금 시작" CTA는 명령형. "함께 시작해요" 같은 동행 톤이 더 학생적.
- **부담감 방지 카피 부재**: 미수행이 떠도 그저 amber 뱃지. "괜찮아요, 지금 이어가요" 같은 어휘가 필요.

### 개선안

**(a) 카피 톤 가이드 4원칙**
1. 명령형(시작/완료) → 청유형(시작해요/완료해요)
2. 평가어("부족") → 관찰어("화요일 짧았어요")
3. 위협("위험") → 권유("쉬어볼까요?")
4. 이모지는 의미 정착된 것만(🙂 컨디션, 🎯 목표, 🔋 번아웃)

**(b) 스트릭 표현 강화**
- 헤더 칩: `🔥 17일 연속` (현재) → `🔥 17일 — 최장 신기록 4일 남음` (목표 visible)
- 7·14·30·100일 마일스톤 도달 시 짧은 confetti motion (200ms) + toast.

**(c) 미수행 카피 재작성**
| Before | After |
|--------|-------|
| 어제 누락분 이월 | 어제 못한 30분, 오늘로 이월했어요 |
| 미수행 | 이월 |
| 위험 신호 — 휴식 권장 | 이번 주 좀 빡빡했어요. 30분 쉬어볼까요? |
| 컨디션 관리 필요해요 | 컨디션 살펴볼게요 |

**(d) "오늘 어땠어요?" 회고 시점 카피**
저녁 21시 이후 또는 마지막 블록 완료 직후, 컨디션 카드가 자동 펼침 + "오늘 어땠어요?" 헤더. 강제 입력 X, dismissible. 자율감 + 회고 습관 유도.

**(e) 동기부여 vs 부담 균형**
- 동기부여: 스트릭, 마일스톤, sparkline 상승, 인사이트 카드.
- 부담 완화: 미수행은 amber이되 "이월"이라는 단어, 임계치 경고는 권유형, 차트는 목표 미달 = 옅음(빨강 아님).

## 7. 온보딩 / 소개 화면

### 현황 (`/planner/onboarding`)
파란 큰 hero 배너 "풀림 플래너 처음이라면" + 단계별 카드 (1. 일/주/월 캘린더로 학습 일정 보기, 2. 다음 블록 hero에서 바로 시작 ...). 각 카드는 좌측에 번호 원 + 본문 + 우측 데모 캡처 이미지. 작은 캡션 글씨가 많고 정보 밀도 높음.

### 문제·기회
- **온보딩이 "기능 설명서"에 가까움**: 학생이 처음 만나는 화면인데 "내 시험을 어떻게 입력하나"의 first step이 없음.
- **데모 캡처가 정적**: 정지 이미지라 인터랙티브 매력 0.
- **번호 1-N 단계가 모두 같은 가중치**: 핵심 3개만 한 화면에 보여주는 게 더 효과적.
- **개인화 진입점이 없음**: "내 시험일 입력 → 첫 학습 블록 자동 생성"의 wow moment가 없다.

### 개선안

**(a) 4단계 점진적 온보딩 (마법사형)**
```
1. 인사 + 목표 시험 선택 (수능 / 모의평가 / 내신 ...)
2. D-day 입력 (자동 D-day 계산 → "오 16일 남았어요" 인사이트)
3. 평일/주말 학습 가능 시간 슬라이더 (예: 평일 18-22, 주말 10-18)
4. 자동 생성된 첫 1일 미리보기 + [이대로 시작] / [조정]
```
각 단계 progress bar 상단(0/4 → 4/4).

**(b) 인터랙티브 데모**
각 단계마다 작은 mock card가 우측에 표시되어, 입력값이 실시간 반영. 학생이 "내가 입력한 것이 화면을 바꾼다"를 체험.

**(c) 첫 학습 블록의 첫 완료 = celebration**
- 온보딩 후 첫 블록 완료 시 confetti(300ms) + toast `첫 블록 완료! 풀림이 학습 패턴을 학습하기 시작했어요`. **이 한 순간이 retention의 90%**.

**(d) "처음이라면" 모드의 in-app 코치 마크**
홈 첫 진입 시 4 spotlight: (1) 일과 그리드 (2) 다음 블록 hero (3) 컨디션 트래커 (4) 회고 카드. 각 1줄 카피 + Skip 옵션.

**(e) 소개 페이지는 "사용 가이드"로 명칭 변경**
온보딩(첫 사용)과 가이드(언제든 찾아보기)는 다른 화면이 되어야 함. 현재 `/onboarding`은 후자에 가까움 → 명칭 `/guide`로, 진짜 onboarding은 첫 로그인 시 별도 마법사로.

## 8. Planner 고유 모션 카탈로그

### 현재 모션
사이드바 hover transition, 헤더 backdrop-blur 정도. 정적 사이트에 가까움.

### 8가지 핵심 모션 (트리거 → 변화 → duration/easing → 효과)

**M1. 현재 시각 라인 자동 흐름**
- 트리거: 시계가 1분 진행
- 변화: 빨간 가로선이 1px씩 부드럽게 아래로 이동, 좌측 라벨 텍스트(`● 14:03`) 숫자 fade-update
- duration / easing: 1000ms / ease-in-out (60초마다 한번)
- 효과: "지금"이 살아 있다는 신호. 자리 잡고 안 봐도 "방금 지나갔구나" 인식.

**M2. 블록 완료 체크 애니메이션**
- 트리거: 진행 중 블록의 [완료] 또는 진행률 100% 도달
- 변화: 카드 stripe가 blue → green 그라데이션 0.3s 전환, 우측 체크 ✓가 0.4s 동안 stroke-draw, 카드 1px 위로 살짝 뜸
- duration / easing: 400ms / cubic-bezier(0.34, 1.56, 0.64, 1) (살짝 overshoot)
- 효과: 보상감. 다음 블록 카드로 자연스러운 시선 이동.

**M3. 컨디션 슬라이더 햅틱한 모션**
- 트리거: 5단 이모지 중 하나 탭
- 변화: 탭된 이모지 1.15배 확대 + 살짝 회전(-5°→0°) bounce, 다른 이모지는 opacity 0.5로 fade, 0.6초 후 자동 fold + toast `🙂 저장됨`
- duration / easing: 250ms / spring(stiffness 400, damping 22)
- 효과: 모바일에서 햅틱 피드백처럼 느껴짐. "선택했다"가 분명.

**M4. D-day 카운트다운 인터랙션**
- 트리거: D-day 칩 hover (데스크) / 탭(모바일)
- 변화: 칩이 가로로 펼쳐지며 `D-16` → `D-16 · 6월 모의평가 · 5/24(토)` 진행, 0.18s. 다시 닫힐 때 0.12s
- duration / easing: 펴짐 180ms ease-out, 닫힘 120ms ease-in
- 효과: 정보 밀도 ↑이면서 공간 차지 ↓. 클릭하면 마일스톤 페이지로.
- D-3 이내일 때 + pulse 모션: opacity 0.8 ↔ 1.0, 2초 주기 ease-in-out infinite.

**M5. 캘린더 스크롤 sticky 헤더 거동**
- 트리거: 일과 카드 내부 스크롤 (모바일 시간 그리드 스크롤 가능 가정)
- 변화: 시간대 그룹 헤더("저녁 코어")가 스크롤 진입 시 상단에 sticky로 고정, 다음 그룹 헤더 진입 시 부드럽게 위로 밀려나감
- duration / easing: native sticky + 16ms throttle
- 효과: 학생이 "지금 어느 시간대 보고 있는지" 항상 인식. 긴 학습 시간 그리드에 필수.

**M6. 빈 슬롯 affordance fade-in**
- 트리거: 빈 슬롯 hover (데스크)
- 변화: 슬롯 배경 transparent → slate-50, 중앙에 `+ 15분 회복` `+ 25분 보강` 두 ghost 버튼이 8px 아래에서 위로 슬라이드 + fade-in
- duration / easing: 150ms / ease-out
- 효과: 빈 시간이 "활용 가능한 자원"으로 보임.

**M7. 인사이트 카드 stream-in**
- 트리거: 리포트 페이지 진입
- 변화: 인사이트 카드 3-5개가 좌→우로 50ms 시차로 fade-in + translateY(8px → 0)
- duration / easing: 카드당 300ms ease-out, 50ms stagger
- 효과: "방금 분석되어 도착한" 인상. 데이터가 살아 있다.

**M8. 스트릭 마일스톤 confetti**
- 트리거: 7/14/30/100일 도달 또는 첫 블록 완료
- 변화: 화면 중앙에서 작은 입자(8-12개) 300ms 동안 방사상 spread + 헤더 스트릭 칩이 1.3배 bounce + toast `🔥 17일 연속 — 최장 신기록!`
- duration / easing: 입자 300ms ease-out, 칩 bounce spring(stiffness 500, damping 18)
- 효과: 학습이 게임화되지만 과하지 않게. 한 번 보고 끝나는 보상.
