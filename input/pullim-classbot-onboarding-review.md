## 0. 한 줄 평

**구조와 정보는 잘 잡혀 있다. 다만 (1) 폰트 사이즈/타깃 사이즈/H 위계가 가독·접근성 하한선 아래에 있고, (2) 카피가 학생 화면임에도 교사 관점 명사구 중심이라 "내가 지금 무엇을 해야 하는지"가 3초 안에 잡히지 않는다.** 디자인을 갈아엎을 필요는 없고, 토큰과 카피 한 번 정비하면 1군 SaaS(Toss/채널톡/Linear) 옆에 두어도 어색하지 않은 수준이 된다.

## 2. 우선순위별 개선 사항

### 🔴 CRITICAL (이번 sprint 안에 머지)

**C1. H1 두 개 + 헤딩 점프 (a11y/SEO)**

- 데모 카드 안 "수학이 형"이 `<h1>`로 마크업됨 → 데모 카드 내부 텍스트는 `<div role="img" aria-label="...">` 또는 `<figure>` + `<figcaption>`으로 감싸고, 내부 헤딩은 시각적으로만 큰 텍스트(`<p>` + 큰 폰트 토큰)로 처리.
- "Scope Guard" `<h4>` → 페이지 H2/H3 흐름에 맞춰 `<h3>` 또는 헤딩이 아닌 강조 텍스트.
- 결과: 한 페이지에 H1 1개, H2 4개(스텝), H3는 없거나 데모 내부에만.

**C2. 모바일 터치 타깃 < 44px (Apple HIG 위반)**

- 4개 인터랙티브 요소가 전부 미달:
  - 최종 CTA "풀림 클래스봇 시작하기" → 패딩 키워 **최소 48px** 높이로. 폰트 14 → 15–16, vertical padding 10 → 14.
  - "봇 채팅 열기" → 동일하게 36–40px 이상, 좌우 폭은 자동.
- `hitSlop` 또는 `padding`으로 시각 사이즈는 유지하면서 hit area만 늘리는 것도 가능하지만, 모바일 CTA는 그냥 시각 사이즈를 키우는 게 맞다.

**C3. 9–10px 텍스트 다수**

- "L1/L2/L3/L4/L5" 칩(9px), "현재 L3"(9px), "자동 스위치: 18:00–19:00 …"(9px), "교사 설정 — Scope Guard"(10px), "실제 봇 헤더(다크 그라데이션)"(10px) 등.
- 토큰화: 칩 최소 11–12px, 캡션 최소 12px, 본문 14–16px. 한국어는 9px에서 자모 식별이 거의 불가.
- 데모 카드를 "이건 예시다"라는 의미로 줄였다면, 카드 자체를 키우는 게 맞지 글자만 줄여서 정보를 숨기지 말 것.

**C4. `maximum-scale=5` 제거 (WCAG 1.4.4 — 텍스트 크기 조절)**

- `<meta name="viewport">`에서 `maximum-scale`, `user-scalable=no` 모두 삭제 권장. `width=device-width, initial-scale=1`만 남길 것. 시각 약자가 핀치로 확대할 권리를 잃는다.

### 🟠 HIGH (다음 release 안에)

**H1. 학생 화면인데 카피가 교사 중심**

- 페이지 hero: "범용 AI 챗봇과 다름 — 교사가 만들고 통제하는 AI 학습 교실"
  - 명사구로 끝나고("…다름") "학생이 무엇을 얻는가"가 비어 있다.
  - 두 마스터 프롬프트 `korean_ux_writing_principles`의 "사용자 입장 + 결과 + 시간/노력 감소" 원칙 위반.
- 스텝 H2도 전부 교사 입장 명사구: "봇 정체성", "교사가 권한 통제", "실시간 모니터링 — 투명성".
- 권장 Before → After 는 §3에 정리.

**H2. 시각 위계 부족 — H2와 본문이 같은 16px**

- H2 16/700 ↔ 본문 16/400. 굵기만 다르고 사이즈가 같다. 빠르게 스캔이 안 된다.
- 권장 토큰:
  - Hero H1 데스크톱 30–32 / 모바일 24–26
  - 섹션 H2 데스크톱 20–22 / 모바일 18–20
  - 본문 16 (모바일), 14는 캡션 한정
  - 칩/배지 12–13 (현재 9–10에서 인상)

**H3. "4분 가이드"라고 약속만 하고 진행감 없음**

- 헤더 배지 "4분 가이드"는 시간 약속이지만, 4개 스텝 중 어디까지 왔는지 보이는 인디케이터가 없다.
- `master-prompt-dashboard-ko.md` §14 onboarding은 "**상단에 진행바**"를 명시. 좌측 사이드바 또는 상단에 sticky 1/4 → 4/4 진행 표시 추가.
- 스크롤형 가이드라면 최소 sticky "1 of 4" 칩 + 각 스텝 카드 좌상단의 번호 원형 안에 connector line(아래로 이어지는 세로선)을 그어 시퀀스를 시각화.

**H4. 데모 카드에 "이건 데모"라는 시그널이 약함**

- "제출하기" 버튼이 `disabled`만 걸려 있다 → 사용자가 누르려다 안 눌리는 죽은 인터랙션.
- 각 데모 카드 우상단에 `<Badge>예시</Badge>` (또는 "Preview") + 카드 전체에 살짝 dashed border 또는 톤다운된 워터마크 배경으로 "이건 실 작동이 아님"을 명확히.
- 부차 캡션 4개 — "실제 봇 헤더 (다크 그라데이션)", "Scope Guard + 시간대 자동 스위치", "실제 라이브 퀴즈 카드", "교사 대시보드 — 실시간 봇 질문 피드" — 모두 디자이너 노트 같다. **삭제 또는 사용자 가치 카피로 교체** (예: "선생님이 직접 만든 봇임을 한눈에 — 이름·과목·범위가 헤더에 표시돼요").

**H5. 종결 CTA가 흐름의 클라이맥스가 되지 못함**

- "준비됐어요" 섹션은 H2 + 한 줄 + 작은 링크 버튼(188×40). 이전 4개 카드의 시각 무게에 묻힌다.
- 풀폭 배너(좌우 그라데이션, 상단/하단 큰 패딩 32–48), 큰 1차 CTA (48–56px 높이, 18px 폰트), 보조로 "둘러보기" 또는 "처음으로" 링크.
- 라벨도 "풀림 클래스봇 시작하기"보다 행동 중심 짧은 동사구가 낫다 → "지금 봇과 대화해 보기" 또는 "내 첫 봇 만나러 가기".

### 🟡 MEDIUM (다음 다음 sprint)

**M1. 인용 부호 표기 통일** — `"수학이 형"`이 직선 따옴표 `"`. 국립국어원 표기는 한글 큰따옴표 `"…"` 또는 작은따옴표 `'…'`. 디자인 시스템에 `<Quote>` 또는 ESLint 룰로 강제. UX 라이팅 마스터 프롬프트의 "한국어 특수성" 항목.

**M2. 헤더의 "17일째" 마이크로카피 모호함** — 무엇이 17일째인지 라벨이 없다. "사용 17일째", "학기 17일째" 등으로 명사화하거나 호버 툴팁/aria-label로 의미 보강. 단독 숫자+조사 패턴은 시계열 데이터에는 OK지만 헤더 영구 노출 자리엔 맞지 않는다.

**M3. 불릿 라인 높이 1.33 → 1.5–1.6** — 12px에 line-height 16px는 너무 빡빡. 한국어 본문은 한자/한글 자형이 영문보다 무겁기 때문에 영문보다 더 넉넉히 줘야 읽힌다. 권장 line-height 1.55–1.7.

**M4. 라이브 퀴즈 데모 — 객관식 보기에 KaTeX/MathJax 없는 LaTeX 노출**

- "f(x) = x³ − 6x² + 9x + 1 의 극댓값과 극솟값의 합은?" — 거듭제곱이 유니코드 위첨자(³, ²)로만 표현. 실제 제품에서 KaTeX가 들어가면 화면이 완전히 달라진다. 데모와 실 제품 간 인지 차이를 줄이려면 데모도 동일 렌더러 사용 또는 명시적으로 "수식 렌더링은 실제 화면에서 더 깔끔하게 표시돼요" 캡션.

**M5. 색·아이콘만으로 상태 전달 금지** — L3 칩은 outline 굵기로만 현재 레벨 표시. 색맹·저시력 사용자에겐 식별 어려움. **체크 아이콘 + "현재" 라벨 동반**으로 색 외 시그널 추가.

**M6. 좌측 사이드바 "봇 찾기 [disabled]"** — 비활성 메뉴가 메인 내비게이션에 그대로 노출. "준비 중" 배지 또는 본 페이지에서 숨김 처리. 사용자가 disabled를 만나면 신뢰가 깎인다.

### 🔵 LOW (점진 개선)

- **L1.** 본문 가독성: 모바일 본문 14 → 15–16px 상향 (양산형 SaaS 표준 매칭).
- **L2.** 헤더 배지 "소개하기 / 4분 가이드" → 브레드크럼이 이미 "소개하기"를 말하고 있어 중복. "4분 가이드"만 남기는 게 깔끔.
- **L3.** 스텝 1의 hero 카드가 단색 다크 블루로 강조되어 있어 학습자가 "1번이 가장 중요한가?"로 오인 가능. 4개 스텝의 시각 강도를 균질화하거나 진행 상태에 따라 강조 위치를 옮길 수 있게.
- **L4.** 푸터 모바일 5-아이콘 탭바가 본문과 너무 가깝다. `padding-bottom: env(safe-area-inset-bottom)`와 상단 24px 여백 보강.

## 4. 추가로 제품팀이 결정해야 할 것

1. **온보딩 = 1회성인가, 영구 참고문서인가?** 현재 사이드바에 영구 메뉴로 노출되고 본문에도 "막히면 언제든 돌아올 수 있어요"라고 쓰여 있음 → 둘 다 의도라면 OK지만, 그렇다면 페이지 이름을 "소개하기"보다 "시작 가이드" 또는 "사용법"이 사용 의도와 더 맞는다.
2. **"4분 가이드"의 4분 기준은?** 측정해 본 적이 있다면 그대로 두고, 아니면 빼는 게 안전. 약속한 시간은 신뢰 자원이다.
3. **데모 카드 인터랙티브 vs 정적**: "라이브 퀴즈"에서 보기를 실제로 선택해 볼 수 있게 한다면 첫 가치 경험("Aha moment")이 한 스크롤 안에 들어온다. 비용 대비 효과가 큰 후보.
4. **모바일 우선 점검**: 화면 캡처 보면 모바일 1뷰포트에 hero + 스텝1 도입부만 들어가고 나머지는 전부 스크롤 아래. 모바일에서 4분은 길게 느껴진다. 모바일은 hero 카드를 더 짧게(120–160px 안쪽) 압축.

## 5. 머지 전 체크리스트 (그대로 복사 가능)

- [ ] 페이지에 `<h1>`이 1개뿐이고, 헤딩 순서가 H1→H2→(H3)로만 진행
- [ ] 모든 인터랙티브 요소가 모바일 ≥44 × 44 px
- [ ] 본문 ≥14px, 캡션/칩 ≥12px (9–10px 텍스트 0개)
- [ ] `<meta viewport>`에서 `maximum-scale` / `user-scalable=no` 제거
- [ ] 데모 카드에 "예시" 배지 추가, 디자이너 캡션 4종 삭제
- [ ] "준비됐어요" 섹션이 풀폭 CTA로 강조됨 (CTA 높이 ≥48, 폰트 ≥16)
- [ ] Hero/스텝 H2/CTA 카피가 §3 사전대로 학생 관점으로 교체됨
- [ ] 인용 부호가 한글 따옴표 `"…"` 또는 `'…'`
- [ ] "봇 찾기 [disabled]" 메뉴 처리 — 숨김 또는 "준비 중" 배지
- [ ] 색 외에 상태 전달 (L3 강조에 아이콘 또는 "현재" 라벨)
- [ ] 진행 인디케이터 (1/4 → 4/4) 추가 또는 스텝 카드 좌측 connector line
- [ ] Reduced motion으로 페이지 진입 시 모든 모션 비활성/단축 확인
