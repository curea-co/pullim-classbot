## 1. AI 명령지침

- "성적뿐 아니라 감정을 본다"는 핸드오프 § 1.4 V5 핵심 가치. **리포트와 감정 체크인은 별 기능이 아니라 한 시스템.** 분리해서 설계하지 말 것.
- **감정 데이터는 민감 정보**. 핸드오프 § 13.1에 따라 교사·학부모 외 열람 금지. 학원 관리자도 익명·집계만.
- **3자 알림 트리거** (학생 + 학부모 + Wee센터)는 자살·자해·우울 키워드 한정. 평범한 "힘듦"은 교사 단독 알림.
- 학부모 발송은 반드시 교사 승인 게이트 통과 (24-48H 자동 승인 정책은 § 5.4 참조).
- 추출본 정책: 클래스봇 도메인 내부에서 자급. 외부 도메인(플래너·Q) 의존 금지. 외부 시스템(Wee센터 API · 카카오 BIZ MESSAGE)은 v2 마일스톤이므로 추출본에서는 mock 인터페이스만.

## 2. 제품 정의

### 2.1 Problem Statement
교사가 매 학생의 학습량·정답률·감정을 일일이 확인하기 어렵고, 학부모는 월말 성적표로만 자녀를 본다. 번아웃·우울 신호는 늦게 드러나서 개입 시점도 평균 1개월 지연. 학생 본인도 자신이 어떤 상태인지 객관화할 도구가 없다.

### 2.2 Product Goal
- **G1**: 학생 본인의 자기보고(감정 체크인)를 매일 30초 안에 수집
- **G2**: 6종 리포트가 자동 생성되어 교사·학부모가 학생 상태를 **3분 안에** 파악
- **G3**: 번아웃 임계 초과 시 **72시간 내** 개입 (KPI § 10.2 — 90% 감지율)
- **G4**: 학부모 리포트 열람률 80% (자녀 학습 가시화 → 재계약/리텐션 견인)

### 2.3 Persona별 영향

| 페르소나 | 이 기능에서 달라지는 것 |
|---|---|
| **P1 교사** | 매일 아침 어제 자동 생성된 학급 리포트 3분 검토 → 위기 학생 1:1 진입 |
| **P2 학생** | 매일 봇 채팅 진입 시 30초 감정 체크인 → 본인 주간 리포트로 자기 관찰 |
| **P3 학부모** | 주 1회 카카오 BIZ MESSAGE로 자녀 주간 리포트 수신 (열람률 80% 목표) |
| **P4 학원 관리자** | 월간 학원 리포트 자동 생성 → 원장 전달 (Flow E) |

## 3. 핵심 기능 정의

### 3.1 MoSCoW

#### Must (MVP)
- [M1] 6종 리포트 목록 화면 — 종류·상태·승인 대기 필터
- [M2] 리포트 상세 — 8 KPI · 1줄 요약 · 위기 신호
- [M3] 학생 일일 감정 체크인 위젯 — 4 이모지 + 1줄 자유 텍스트(선택)
- [M4] 웰빙 지수 5지표 가중 평균 계산 + 임계 알림
- [M5] 교사 승인 → 학부모 전송 워크플로 (Mock 카카오 BIZ MESSAGE)
- [M6] 학생 본인 주간 리포트 열람 페이지

#### Should (v1)
- [S1] 학부모 자동 승인 (24-48H 무응답 시) + 학부모 수동 승인 옵션
- [S2] 3일 연속 "힘듦" 자동 감지 → 교사 푸시
- [S3] 키워드 게이트 — 자살·자해 키워드 발견 시 3자 알림 (교사+학부모+Wee센터)
- [S4] 리포트 PDF 다운로드 (학부모 이메일 첨부 대안)
- [S5] 학급 비교 리포트 (학원 관리자 시점)

#### Could (v2)
- [C1] Wee센터 API 직접 연동 (정부 기관 연동)
- [C2] 학부모 자녀 데이터 동의 워크플로 (만 14세 미만 법정대리인 동의)
- [C3] 분기별 학원 전체 학습 분포 비교 + 트렌드

#### Won't (현 단계 제외)
- 감정 데이터를 학원 관리자에게 개별 단위로 노출 (집계만)
- AI가 학생의 감정을 일방적으로 판정 (반드시 자기보고 1차 + AI 보조 2차)
- 학부모가 학생의 봇 대화 원본 열람 (요약만)

#### 3.3.3 학생 웰빙 허브 (`/classbot/wellness`)
| 구성 | 설명 |
|---|---|
| 헤더 | "오늘 어땠어?" + 오늘 체크인 완료/미완료 상태 |
| 웰빙 지수 카드 | 0~100 점수 + 7일 추세 + 5지표 분해 (§ 9.2 5지표 펼침 패턴) |
| **담당 봇 코멘트 카드** | **가장 점수 낮은 영역의 담당 봇이 자동으로 한 줄 + actionable CTA**. 봇 시그니처 컬러 좌측 라이너. 예: "🧑‍🏫 수학이 형 · 19:55  /  '오늘 그럭저럭이었구나. 6일째 출석! 내일 1문항만 같이 풀어볼까?'  [좋아 → 1문항]" (§ 9.3 참조) |
| 오늘 체크인 CTA | `[30초 체크인 시작]` (미완료 시) / `[다시 작성]` (완료 후 1회) |
| 주간 감정 그래프 | 7일 × 4이모지 점 도표 + 7일 막대(웰빙 지수) — § 9.1 컬러 매핑 |
| "선생님이 곁에 있어" 카드 | 위기 임계 근접 시 노출 — 1:1 면담 신청 / 봇 대화 CTA |
| 본인 리포트 진입 | `[내 주간 리포트 보기]` → `/classbot/me/report` |

> 회귀 사례 (private-classbot.md § 6.2): 웰빙은 봇이 개입하는 영역인데 봇 아바타·이름이 페이지에 한 번도 안 나옴. 담당 봇 코멘트 카드로 시각화.

## 4. UX Flow

### 4.1 Flow R1 — 학생 일일 감정 체크인 (매일)
```
1. 학생이 19:00 봇 채팅 진입 시도
2. 오늘 체크인 미완료 → 부드러운 인터셉트 모달: "오늘 어땠어? (30초)"
3. 4이모지 중 "😐 그저그럼" 선택
4. 강도 3 슬라이더 (스킵 가능)
5. "수업 좀 어려워서..." 한 줄 입력
6. 완료 → "기록했어. 내일 또 봐."
7. 봇 채팅 정상 진입
```

### 4.2 Flow R2 — 일일 리포트 자동 생성 (교사 시점)
```
1. 19:50 — 도함수 활용 수업 종료
2. 시스템: rep_c 자동 생성 (status: 'draft')
3. 20:00 — 일일 학생별 리포트 6건 자동 생성 (status: 'pending-approval')
4. 다음날 09:00 — 교사 홈 "발송 대기 6건" 카드
5. /teacher/reports 진입 → 도현 리포트 우선 검토 (위기 인디케이터)
6. [수정 후 승인] — 1:1 면담 메모 첨부 확인 후 발송
7. 학부모 휴대폰에 카카오 BIZ MESSAGE 도착
```

### 4.3 Flow R3 — 번아웃 조기 경보 (핸드오프 Flow D)
```
1. 도현 — 3일 연속 "힘듦" 체크인 + 정답률 -15% + 학습 시간 -40%
2. 시스템: 웰빙 지수 48 → 임계 60 미달 + 3일 지속
3. 교사 푸시 알림: "도현이 신경 쓰일 신호 있음"
4. 리포트 센터 상세 진입 → 위기 신호 패널 점등
5. [1:1 상담 시작] → 봇 채팅에 교사가 학생 1:1 모드 진입
6. 또는 [Wee센터 연결] → 학부모에게 요약 (민감 정보 제외) 전송
7. 시스템: 개입 시각 기록 → 72h 내 후속 체크 알림
```

### 4.4 Flow R4 — 자살 키워드 3자 알림 (긴급)
```
1. 학생 감정 체크인 자유 텍스트에 "사라지고 싶다" 입력
2. 키워드 게이트 즉시 발동 — 학생 화면에 부드러운 모달:
   "지금 도움이 필요해 보여. 선생님과 상담사가 같이 도와줄 거야."
3. 동시에 백그라운드 3자 알림:
   - 교사 — 정확한 텍스트
   - 학부모 — 요약 ("자녀가 어려운 마음을 표현했어요. 곧 연락드릴게요.")
   - Wee센터 — 익명 코드 + 지역 정보 (v2 실 API 연동, P0은 mock log)
4. 학생에게 즉시 봇이 침착한 톤으로 1:1 대화 시작 — Scope L5 전환
5. 교사 도착 후 봇 → 교사 핸드오프
```

### 4.5 Flow R5 — 학부모 주간 리포트 (Flow A에 연결)
```
1. 일요일 18:00 — 윤서 학부모 주간 리포트 자동 생성 (rep_a)
2. 교사 승인 워크플로 → 자동 또는 수동 승인
3. 24h 자동 승인 (학원 정책에 따라 0~48h 설정 가능)
4. 학부모 카카오 BIZ MESSAGE 도착
5. 학부모 PDF 첨부 다운로드 또는 웹 링크로 상세 열람
6. 학부모 회신("우리 아이 잘하고 있죠?") — 교사 채널로 전달
```

| 자살 키워드 3자 알림 수신 | — | ✅ | — | — | ✅ 요약 |

## 5. 운영 로직

### 5.1 웰빙 지수 계산 (핸드오프 § 7.3)
```
wellbeing = w1 × studyTimeRatio
          + w2 × accuracyTrend7d
          + w3 × sessionCompletionRate
          + w4 × emotionCheckInScore
          + w5 × botInteractionBalance

기본 가중치: w1=20, w2=20, w3=15, w4=30, w5=15  (합 100)
```
- 임계: `wellbeing < 60` 3일 지속 → 교사 알림
- `wellbeing < 40` 1일이라도 → 즉시 교사 푸시
- 가중치는 봇 빌더 § 7 (안전) 단계에서 교사 조정 가능

### 5.2 키워드 게이트
화이트리스트 기반 정규식 매치 (사전 정의):
- 자살·자해 카테고리 (5단계 강도)
- 우울·무기력 카테고리 (3단계 강도)
- 학교 폭력 카테고리 (별도 처리 — Wee센터 대신 학교 상담교사)

매치 시:
- 학생 화면에 즉시 부드러운 인터셉트 (점수·진단 단어 사용 금지)
- 3자 알림 (강도 4+에서만 — false-positive 방지)
- 봇이 자동 1:1 대화 시작 (Scope L5)

### 5.3 리포트 상태 전이
```
draft ──자동 생성──> pending-approval ──교사 승인──> approved
                                          └──24h 무응답──> approved (자동)
approved ──학부모 발송──> sent
```

### 5.4 학부모 자동 승인 정책
| 학원 정책 | 자동 승인 시한 |
|---|---|
| 보수 (수동만) | 0h — 교사 명시 승인 필수 |
| 표준 | 24h |
| 적극 (자동 발송) | 즉시 (단, 위기 신호 있는 리포트는 무조건 수동) |

봇 빌더 § 7에서 학원·교사가 설정.

### 5.5 검증 규칙
| 항목 | 제약 |
|---|---|
| 감정 체크인 자유 텍스트 | 0 ≤ length ≤ 200자 |
| 리포트 1줄 요약 | 30 ≤ length ≤ 200자 |
| 학부모 발송 미리보기 | 길이 무제한, 단 카카오 BIZ 템플릿 형식 검증 |
| 위기 신호 알림 무시 | 12h마다 재알림 (해결 표시 전까지) |

### 5.6 데이터 보존
- 감정 체크인 원문: 90일 후 자동 삭제 (학생 동의 시 보존 가능)
- 리포트: 학기 종료 후 1년 보존 → 자동 익명화
- 3자 알림 로그: 영구 보존 (법적 기록)

## 8. 브랜딩 / UX Writing

### 8.1 톤 원칙
- **학생 대면**: 따뜻하고 비판단적. "힘듦"이라는 단어를 함부로 노출하지 않음. 점수·진단·낙인 단어 금지.
- **교사 대면**: 정확·간결·행동 유도. 위기 신호는 분명히 표시.
- **학부모 대면**: 안심·정보 균형. 부정적 사실은 "함께 도울 방법" 프레임.

### 8.2 마이크로카피

| 상황 | 카피 |
|---|---|
| 학생 체크인 인터셉트 | "오늘 어땠어? 30초만 짚고 가자." |
| 4이모지 라벨 | 😄 좋아 · 🙂 그럭저럭 · 😐 그저그래 · 😔 힘들었어 |
| 체크인 완료 | "기록했어. 내일 또 봐." |
| 키워드 게이트 모달 | "지금 마음이 무거워 보여. 도와줄 사람이 곁에 있어." |
| 위기 학생 교사 알림 | "도현이 — 신경 쓸 신호 3개. 지금 한 번 봐주세요." |
| 학부모 리포트 발송 안내 | "주간 리포트를 보낼 준비가 됐어요. 24시간 안에 자동 발송돼요." |
| 자동 승인 임박 | "12시간 후 자동 발송돼요. 수정할 게 있으면 지금 해주세요." |
| 학생 본인 리포트 헤더 | "이번 주의 나" (절대 "성적표"라 부르지 않음) |
| 3일 연속 힘듦 후 학생 진입 | "이번 주 좀 무거웠지. 같이 가볍게 가보자." |

### 8.3 금지어 (학생 가시 영역)
- "번아웃", "우울", "위기", "임계" — 라벨로 노출 금지
- "낮은 점수", "부족" — "더 해보자", "신경 쓸 부분"으로 교체
- "AI가 분석한 너의 상태" — "내가 본 너의 한 주"로 교체 (봇이 1인칭)

## 9. 디자인 시스템

### 9.1 컬러 토큰

#### 9.1.1 이모지 4단계 (학생 자기보고)
- 동일 hue, 채도만 다른 4단계 (낙인 방지 — 빨강·회색 같은 강한 대비 금지)

#### 9.1.2 웰빙 게이지 / 7일 막대 컬러 매핑

회귀 사례 (private-classbot.md § 6.2): 7일 막대가 모두 `brand.600` 단조라 좋은 날/나쁜 날 trend 파악 불가.

| 점수 구간 | 토큰 | 컬러 |
|---|---|---|
| 0–40 | `danger.bg` (`#FCE9EA` bg + `danger.fg #C03B3F` 상단 dot) | 위험 — 학생 화면에도 부드럽게 노출 가능, 라벨은 "신경 쓸 부분" |
| 41–60 | `warning.bg` (`#FFF7E6` bg + amber dot) | 주의 |
| 61–80 | `brand.300` (`#8BAEFF`) | 보통 |
| 81–100 | `success.fg` (`#0E8C56`) | 양호 |

룰:
- **학생 노출 막대**: 위 4구간 컬러 매핑으로 trend 시각화 — 단, 0–40 막대 위에 "위기"·"위험" 같은 라벨 노출 금지 (§ 8.3 금지어).
- **교사 화면**: 동일 컬러 매핑 + "위기 학생" 라벨 노출 가능.
- 데이터 부재 막대: `border.subtle` 점선 outline.

#### 9.1.3 위기 알림 패널
- `bg-pullim-slate-900` + `text-pullim-lemon` (이미 [/teacher/classbot/page.tsx](src/app/(teacher)/teacher/classbot/page.tsx#L55-L63) 패턴 재사용)
