## 2. Brand Personality

### 2.1 핵심 성격 (5가지)

| 성격 | 의미 | 반대 (피해야 할 것) |
|------|------|------------------|
| **친근한** | 학생 옆에 있는 형·누나 같은 톤 | 차갑고 사무적 |
| **전문적** | 교육과정·IRT 등 데이터 기반 신뢰 | 어설픈 일반론 |

- **나이**: 20대 후반 형/누나 (학생 입장에서 살짝 윗사람)
- **성격**: 똑똑하지만 뽐내지 않음, 학생 편에서 같이 고민
- **말투**: 친근한 존댓말, 가끔 이모지 (과하지 않게)

## 3. Tone & Voice

### 3.1 톤 (Tone) — 상황별

| 상황 | 톤 | 예시 |
|------|---|------|
| 일반 안내 | 친근·중립 | "오늘 어떤 문제 풀어볼까요?" |
| 격려 | 따뜻·긍정 | "조금만 더 가면 돼요. 잘하고 있어요." |
| 정답 / 성공 | 축하 | "완벽해요! 🎉 다음 문제 가볼까요?" |
| 오답 | 부드럽지만 명확 | "이번엔 아쉽네요. 어디서 막혔는지 같이 봐요." |
| 시스템 오류 | 신뢰·해결책 제시 | "잠시 문제가 생겼어요. 잠시 후 다시 시도해주세요." |
| AI 제한 (Scope L1) | 정중·이유 명시 | "지금은 시험 시간이라 도와드릴 수 없어요. 끝나면 같이 풀어봐요." |

### 3.2 보이스 (Voice) — 절대 원칙

✅ **DO**
- 친근한 존댓말 ("~해요", "~볼까요?", "~돼요")
- 학생 입장에서 같이 고민하는 표현 ("같이", "함께", "우리")
- 짧고 명확한 문장 (한 문장 ≤ 25자 권장)
- 학생을 응원하는 톤 ("잘하고 있어요", "한 번 더 해봐요")

❌ **DON'T**
- 반말 ("야", "~해라", "~다")
- 과한 격식 ("~하시기 바랍니다", "~해주십시오")
- 학생을 가르치려 드는 톤 ("이건 이렇게 해야 해요", "당연히 알아야 해요")
- 부정적 평가 ("이걸 모른다고?", "쉬운 건데")
- 영문 코드명 노출 ("Infinity Engine", "Socrates", "Conqueror") — § 5.1
- 어려운 한자어 (잔존·임박·차시·진척도 등) — § 6 교체 표 적용
- 학술 기호·약어 (θ, σ, IRT 등) — 학생 UI에선 한국어로 풀어 쓴다 (§ 5.2)
- **내부 코드 단독 노출** (Scope `L3`·`L4`, Tier `T2`·`T3` 등) — § 5.3 한글 병기 룰 적용

### 3.3 학생 vs 교사 UI의 톤 차이

| 영역 | 톤 |
|------|----|
| 학생 UI | 친근한 존댓말 |
| 교사 UI | 정중한 존댓말, 약간 더 격식 ("~해주세요", "~하실 수 있어요") |
| 학부모 UI | 정중·신뢰 ("~하시면 됩니다", "자녀분") |
| 시스템 알림 | 중립적 ("~되었습니다", "~필요해요") |

#### 4.6.1 봇별 시그니처 인사 (§ 4.6.2 봇 5종)

봇 페르소나를 시각·모션뿐 아니라 **첫 발화 카피**에서도 차별화한다. 호명 + 봇 특유의 1차 질문.

| 봇 | 시그니처 인사 패턴 |
|---|---|
| **수학이 형** (단정·반말) | "서연 안녕! 수학이 형이야 🙌 오늘 미적분 어디까지 갔어?" |
| **영어 누나** (상냥·존대) | "서연 학생, 안녕하세요. 오늘은 어떤 지문 같이 볼까요?" |
| **과학 쌤** (호기심·반말) | "서연! 오늘 궁금한 거 뭐야? 같이 파보자." |
| **국어 누나** (차분·존대) | "서연 학생, 오늘 어떤 글이 마음에 걸렸나요?" |
| **사회 코치** (격려·반말) | "서연! 오늘도 잘 와줬어. 같이 한 칸씩 가보자." |

#### 4.6.2 봇 5종 시그니처 (브랜드·페르소나 단일 진실원)

| 봇 | 액센트 컬러 | 아바타 배경 | 모티프 | 어조 | 호칭 |
|---|---|---|---|---|---|
| **수학이 형** | `brand.600` + `accent.lime` `#E6FF4C` | navy radial | √ x² ÷ 수식 도트 | 단정·반말 | "형" |
| **영어 누나** | coral `#FF7A6B` | warm-rose | A B C 살짝 회전 | 상냥·존대 | "누나" |
| **과학 쌤** | mint `#22C5A8` | sage radial | 분자/원자 도트 | 호기심·반말 | "쌤" |
| **국어 누나** | violet `#7B5CFF` | lavender | 한글 자모 ㅎㄱㅁ | 차분·존대 | "누나" |
| **사회 코치** | amber `#D97706` | cream | 지도 라인 | 격려·반말 | "코치" |

룰:
- **브랜드 청색은 사이드바·CTA에서 유지**. 봇 액센트는 **말풍선 좌측 라이너·아바타 배경·M2 타이핑 점**에만 적용 ([08-design-system.md § 15](08-design-system.md)).
- 봇 이름은 교사가 빌더에서 변경 가능하지만, **위 5인은 기본 템플릿**으로 카피·디자인 모두 본 표를 권위로 사용.
- 어조(반말/존대) 토글은 학년·관계 설정 기반 (예: 초등=형·누나 반말, 고등=쌤·존대). 봇 빌더에서 결정.

#### 4.6.3 시간대별 인사 룰

호명 다음에 시간대 인사를 자연스럽게 결합. **봇별 첫 발화 시그니처(§ 4.6.1) 앞**에 prefix 가능.

| 시간대 | prefix 카피 (반말 기본) | 존대 변형 |
|---|---|---|
| 06:00–10:59 | "좋은 아침이야, {이름}!" | "좋은 아침이에요, {이름} 학생." |
| 11:00–13:59 | "{이름}! 점심은 챙겼어?" | "{이름} 학생, 점심 챙기셨어요?" |
| 14:00–17:59 | "{이름}, 오늘도 만나서 반가워." | "{이름} 학생, 오후도 화이팅이에요." |
| 18:00–21:59 | "수업 끝났네, {이름}! 고생했어." | "수업 마치셨네요, {이름} 학생. 수고하셨어요." |
| 22:00–01:59 | "{이름}, 늦은 시간까지 수고가 많아." | "{이름} 학생, 늦은 시간이네요. 조심히 끝내요." |

### 4.10 클래스봇 온보딩 (학생 소개하기)

**원칙**: 온보딩 카피는 **학생 입장의 1차 관심사**로 재작성한다. "교사가 관리한다"는 권위적 프레임 대신 "내 봇이 어디까지 도와줘?"라는 학생 시선.

| 옛 카피 (권위·기관 시점) | 신규 카피 (학생 시점) |
|---|---|
| "Scope Guard 5단계 — 교사가 관한 통제" | **"우리 봇은 어디까지 알려줘? — 안전 등급 L1~L5"** |
| "봇 정체성 — 디지털 분신 선생님" | **"내가 만나는 봇은 진짜 우리 선생님 — 디지털 분신이야"** |

### 5.3 Scope / Tier 코드 노출 룰 (클래스봇 전용)

권위 문서(`07_풀림_클래스봇_핸드오프.md` § 4.2·4.3)는 5단계 Scope Guard (L1~L5)와 3-Tier 모델 라우터 (T1~T3)를 **내부 표현**으로 정의한다. 학생 UI에서는 코드 단독 노출 금지.

#### 5.3.1 Scope L1~L5

| 코드 | 의미 (학생 노출 카피) | 학생 첫 등장 패턴 |
|---|---|---|
| L1 | 시험 모드 (응답 차단) | "지금은 시험 모드라 도와줄 수 없어 (L1)" — 코드는 괄호로만 |
| L2 | 경고 (질문은 받음) | "이건 답할 수 없어요" |
| L3 | 개념 모드 (개념·정의·예시) | **"개념까지만 알려주는 모드 (L3)"** |
| L4 | 단계 모드 (풀이 방향) | "풀이 방향까지 알려주는 모드 (L4)" |
| L5 | 답 모드 (정답 포함) | "오답 복습 모드 — 정답까지 같이 봐 (L5)" |

룰:
- **"L3"·"L4" 단독 노출 금지** — 항상 한글 설명 또는 한글 라벨 + 괄호 코드.
- 학생은 모드를 **변경할 수 없다** (권위 문서 § 4.2). 표시 위치는 **봇 헤더 카드 내부**(메타) 또는 **시스템 안내 메시지**에 한정. 입력바 풋터에 매번 노출 금지 ([04 § 9.5](04-ux-flow.md)).
- 첫 등장 시 ⓘ 툴팁으로 "이 봇은 지금 개념 설명까지만 가능해요" 같은 풀어 쓴 설명 제공.

#### 5.3.2 Tier T1~T3 (모델 라우터)

내부 비용·성능 표현. **학생 UI 노출 절대 금지**.

| 코드 | 학생 노출 | 교사 UI 노출 |
|---|---|---|
| T1 (Haiku) | ❌ 없음 — "빠른 응답" | 교사 빌더·리포트에 "T1 (빠른)" 가능 |
| T2 (Sonnet) | ❌ 없음 — "친근 톤" 같은 페르소나 라벨로만 | "T2 (기본)" |
| T3 (Opus) | "AI가 깊게 분석 중이에요. 5초 정도 걸려요." (§ 4.8) — 코드 없이 응답 시간만 | "T3 (심화)" |

> 회귀 사례: 클래스봇 헤더 "친근 톤 · T2" 칩에 코드 노출 — 학생은 T2가 뭔지 모름. "친근 톤"으로만 표시.

## 6. UX Writing — 한자어 · 표현 절감

> **권위**: [proc/research/2026-05-06_hanja-vocabulary-ux-policy.md](../research/2026-05-06_hanja-vocabulary-ux-policy.md)
> 배경(OECD PIAAC, 한국형 실질적 문맹 분석, UI vs 콘텐츠 분리 원칙)은 권위 문서 참조. 본 절은 적용 규칙만.

### 6.1 핵심 원칙 — UI 카피 vs 학습 콘텐츠 분리

| 영역 | 한자어 정책 |
|---|---|
| **UI 카피** (메뉴·버튼·빈 상태·에러·온보딩·툴팁·알림·차트 라벨) | **최소화** — 고등학생이 한 번 읽고 의미가 잡혀야 함 |
| **학습 콘텐츠** (문항·해설·개념 설명·교과서 인용) | **유지** — 시험 환경 정합 우선. 임의 평탄화 금지 |

> 경계선이 흐려지면 "쉬워서 좋다"가 아니라 **"공부에 도움 안 되는 앱"**이 된다. 풀림 도메인명("플래너 / Q / 클래스봇 / 라이브러리")이 한자어 도메인명("학습관리·문제은행")을 의도적으로 피한 결정 자체가 본 원칙의 적용 사례.

### 6.2 한자어 교체 표 (UI 카피)

| 한자어 | 교체안 | 적용 영역 예 |
|---|---|---|
| 잔존 / 잔존률 | 남은 기억 / 기억 정도 | 망각 곡선·복습 카드 |
| 잔여 학습량 | 남은 공부 / 남은 학습 | 진도 카드 |
| 임박 / 임박한 | 곧 / 곧 다가오는 | 일정·알림 |
| 차시 | 회차 | 강의·콘텐츠 카드 |
| 미달 / 미충족 | 부족 / 아직 모자람 | 점수·달성률 |
| 진척도 | 진행률 | 대시보드 |
| 고지 / 공지 | 알림 | 시스템 |
| 회신 | 답장 | 메시지 |
| 익월 / 익일 | 다음 달 / 다음 날 | 일정 |
| 상기 / 하기 | 위 / 아래 | 안내 |
| 종료 | 끝남 / 마침 | 상태 |
| 본인 인증 | 본인 확인 / 내 정보 확인 | 인증 |

> 표는 살아있는 문서 — 회귀 사례 발견 시 행 추가.

### 6.3 불필요한 텍스트·컴포넌트 절감

작성·리뷰 시 의심 항목:
- **장식형 부제**: 본문이 이미 말한 내용을 다시 말하는 부제 → 삭제
- **과잉 배지/태그**: 한 카드에 다른 의미 배지가 3개 이상이면 정보 위계 재설계
- **전문가 톤 압축 카피**: "θ 0.42 → +0.05 변동, σ 0.08" 같은 한 줄 다지표 압축은 분리 또는 요약 문장으로 변환
- **의사결정 무관 메타**: 학생 행동 선택에 영향 없는 메타데이터는 상세 페이지로 이동

### 6.4 콘텐츠 영역 한자어 — 부가가치로 전환 (향후)

학습 콘텐츠(문항·해설)에서 한자어를 빼지 않는 대신, 모르는 한자어를 **학습 자산**으로 만드는 기능을 별도 plan으로 검토:
- 한자어 long-press → 한자 + 음 + 뜻 툴팁
- 어휘 메모리 — 풀림 Q의 메모리 도메인과 결합
- 단어 형태소 분해 — "잔존(殘存)" → 殘(남을 잔) + 存(있을 존)

상세 검토는 [research 문서 § 2.3](../research/2026-05-06_hanja-vocabulary-ux-policy.md) 참조. 본 spec에서는 "콘텐츠는 평탄화하지 않는다"만 강제.

### 6.5 검증 질문

신규/리뷰 화면 카피마다:
- [ ] **고등학생이 한 번 읽고 의미가 잡히는가** (1차 검수 질문)
- [ ] § 6.2 한자어 교체 표 후보를 교체했는가
- [ ] § 5.2 학술 기호·약어를 한국어로 풀었는가
- [ ] 같은 정보를 두 번 말하거나, 의사결정 무관 메타가 쌓이지 않는가
