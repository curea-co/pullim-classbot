# 02. 제품 정의 (Scope Statement / Product Goal)

## 1. Problem Statement

### 한국 EdTech의 구조적 공백

> "AI 생산 + 마켓플레이스 + 학습 분석"이 통합된 플레이어가 부재하다.

| 단계 | 현재 시장의 분절된 솔루션 | 문제점 |
|------|----------------------|------|
| 콘텐츠 생산 | 교사가 한땀한땀 제작 (워드, PPT, 자체 저작도구) | 생산성·품질 한계 |
| 콘텐츠 유통 | 출판사·학원·일부 마켓 (콘텐츠 분산) | 검증·신뢰성 부족 |
| 학습·분석 | 인강·EBS·뤼이드(추천)·콴다(질의응답) | **실시간 생성 불가**, 데이터 분절 |

학생은 4~5개 앱을 옮겨다니며 학습한다. 데이터가 끊기면 개인화가 깊어질 수 없다.

### 해결할 핵심 문제

1. **콘텐츠 부족**: 학생 수준에 맞는 문제·해설·자료가 끊긴다
2. **개인화 한계**: 정답률만 보고 추천 — 풀이 과정·오답 패턴은 못 잡는다
3. **데이터 단절**: 학습·콘텐츠·교사 영역이 따로 돈다 → 플라이휠 부재
4. **교사 부담**: 우수 교사일수록 자료 제작에 시간 낭비, 코칭 시간 부족

## 2. Product Goal

### 비전

> **AI가 만들고, 전문가가 검증하고, 데이터가 증명하는** — 교육 콘텐츠의 생산·유통·소비 통합 플랫폼

### 미션

1. 콘텐츠 생산 비용 1/10 (스튜디오)
2. 학습 효율 2배 (스터디, 하버드 RCT 2025 기준)
3. 교사 수익 3배 (스토어, 1만 크리에이터 목표)

### 풀림 3축 (생산 → 유통 → 소비)

```
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│ 풀림 스튜디오  │──▶│  풀림 스토어   │──▶│  풀림 스터디   │
│  (생산)       │   │  (유통)       │   │  (소비)       │
│              │   │              │   │              │
│ • AI 문제생성 │   │ • 마켓플레이스 │   │ • 적응형 학습  │
│ • 자료 제작   │   │ • 커머스      │   │ • AI 튜터     │
│ • 검증 시스템 │   │ • 크리에이터   │   │ • 진단·분석   │
│ • OCR 변환   │   │ • 구독 서비스  │   │ • 앱 생태계   │
└──────┬───────┘   └──────┬───────┘   └──────┬───────┘
       │                  │                  │
┌──────┴──────────────────┴──────────────────┴──────┐
│     공유 인프라 (CUREA DEEP · 분류DB · 크레딧)      │
├────────────────────────────────────────────────────┤
│   데이터 플라이휠 (학습→개선→품질↑→유입↑→학습)      │
└────────────────────────────────────────────────────┘
```

### 각 축의 역할

- **스튜디오 (생산)**: AI 기반 콘텐츠 창작실 — 크리에이터가 최소 노력으로 최대 품질 생산
- **스토어 (유통)**: 교육 특화 마켓플레이스 — 크리에이터와 학습자 연결, 검증된 콘텐츠만 유통
- **스터디 (소비)**: AI 코칭 학습 플랫폼 — 학생의 개인화 학습 환경

### 공유 기반

- **CUREA DEEP 엔진**: 문제 생성(스튜디오) + 품질 평가(스토어) + 적응형 출제(스터디)
- **3-Depth 교육과정 분류 DB**: 모든 콘텐츠의 메타데이터 표준
- **크레딧 시스템**: 스튜디오 비용 → 스토어 수수료 → 스터디 구독 연동

## 3. 데이터 플라이휠

```
스터디 (학습 데이터: 정답률, 풀이 패턴, 취약점, AI 튜터 피드백)
   ↓
CUREA DEEP (AI 모델 개선: 더 정확한 문제, 더 나은 해설, 더 정밀한 IRT)
   ↓
스튜디오 (콘텐츠 품질↑: 고품질 문제, 정확한 메타데이터, 크리에이터 생산성↑)
   ↓
스토어 (콘텐츠 풍부↑: 더 많은 상품, 실증 효과 리뷰, 크리에이터 수익↑)
   ↓
스터디 (사용자↑: 더 나은 적응형, 더 나은 코칭, 더 많은 학습자 유입)
   ↓ 루프 반복
```

## 4. Persona

### 4.1 1순위: 학생 (Learner) — 고등학생

| 속성 | 내용 |
|------|------|
| 학년 | 고1~고3, 재수생 |
| 동기 | 수능·내신 점수 향상 |
| 디바이스 | **태블릿 (가로/세로) 메인**, 모바일·데스크탑 보조 — 태블릿 가로는 데스크탑 비율로, 세로는 모바일 비율로 자연 확장 (확장성 우선 전략) |
| 갈증 | "내 수준에 딱 맞는 문제 무한 공급", "왜 틀렸는지 깊게 알고 싶음" |
| 두려움 | 학습 중단(앱 옮겨다니기), 시간 낭비, 비효율 |
| 핵심 행동 | 진단 → 풀이 → 해설 → 정복 → 갱신 (6단계 루프) |

#### Mock 페르소나
- **김수학** (고2): 수학 약점, 모의고사 3등급 → 1등급 목표
- **수학이 형** (재수): 정시 준비, 시간 관리 핵심
- **서연·하윤·도현** (고2-A반): 클래스봇 활용 그룹

### 4.2 2순위: 교사 (Manager) — 학원·과외·학교

| 속성 | 내용 |
|------|------|
| 역할 | 강사, 원장, 학교 교사 |
| 동기 | 수업 효과↑, 학생 관리 부담↓, 추가 수익 |
| 디바이스 | 데스크탑 우선 (운영) |
| 갈증 | "수업 중 학생 상태 실시간 파악", "콘텐츠 빠르게 만들고 검증" |
| 두려움 | AI에게 일자리 대체, 학생 데이터 통제권 상실 |
| 핵심 행동 | 봇 빌드 → 수업 운영 → 리포트 → 환류 |

### 4.3 3순위: 학부모 (Assistant)

| 속성 | 내용 |
|------|------|
| 동기 | 자녀 성적·학습 상태 가시화 |
| 갈증 | "내 아이가 진짜 공부하고 있는가", "어디가 약한지" |
| 권한 | 자녀 리포트 열람만, 상세 분석은 교사 통해서만 |

### 4.4 4순위: 기관 관리자 (Owner)

| 속성 | 내용 |
|------|------|
| 동기 | 학원·학교 운영 효율, 강사 관리 |
| 권한 | 전체 설정 + 보고서 접근 |

## 5. 시장 타당성

| 항목 | 기준 |
|------|------|
| 시장 규모 | 한국 EdTech ₩6.2B (2025), SAM ₩1.5B (K-12 AI 학습) |
| 경쟁 공백 | "AI 생산 + 마켓플레이스 + 학습" 통합 플레이어 없음 |
| 정책 타이밍 | 교육부 AI 디지털교과서 (2025~), 디지털 교실 투자 (2026) |
| 기술 성숙도 | AI 튜터링 학습 효과 2배 실증 (하버드 RCT, 2025) |

### 경쟁사 대비 우위

| 영역 | 경쟁사 | 풀림 우위 |
|------|------|---------|
| IRT 적응형 | 뤼이드 | 풀림: **실시간 생성 가능** (뤼이드 불가) |
| 질의응답 | 콴다 | 풀림: **3-Depth 교육과정 바인딩** + 학습 분석 |
| 콘텐츠 마켓 | 출판사·학원 | 풀림: **AI 검증 + 실증 효과 데이터** |
| 인강 | 메가스터디·EBS | 풀림: **개인화·실시간성** |

## 6. 도메인 확장성 (장기 비전)

수능·내신을 넘어 동일 아키텍처로 확장:
- 자격증 (KICE 인증, 의·약·법학 PEET)
- 어학 (TOEIC, TEPS, JLPT)
- 코딩 교육 (백준·릿코드 형태)
- 기업 교육 (CFA, PMP)
- 글로벌 (SAT, IB, AP)

**기술적 근거**: 어댑터 JSON 1개로 도메인 전환. 코드 변경 0줄 목표.
